## 1️⃣ Climbing Stairs

### 📖 Description 

Tujhe `n` steps climb karne hain. Tu ya toh **1 step** ya **2 steps** ek baar mein le sakta hai. **Kitne unique ways** hain top tak pahunchne ke?

**Example:** `n = 3` → Ways: (1,1,1), (1,2), (2,1) → Output: `3`

### 💡 Concept

```
Ways(n) = Ways(n-1) + Ways(n-2)
Base Cases: Ways(1) = 1, Ways(2) = 2
```

Yeh **Fibonacci sequence** jaisa hai!

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    public int climbStairs(int n) {
        // Memoization array: -1 means not computed yet
        int[] memo = new int[n + 1];
        Arrays.fill(memo, -1);
        return solve(n, memo);
    }
    
    private int solve(int n, int[] memo) {
        // Base cases
        if (n <= 2) return n;
        
        // If already computed, return stored value
        if (memo[n] != -1) return memo[n];
        
        // Recursive relation: ways to reach n = ways to reach (n-1) + ways to reach (n-2)
        memo[n] = solve(n - 1, memo) + solve(n - 2, memo);
        return memo[n];
    }
}
```

### ✅ Time Complexity: O(n) | Space: O(n)

## 2️⃣ Min Cost Climbing Stairs

### 📖 Description 

Har step `i` par chadhne ka `cost[i]` hai. Tu ya toh step `0` ya step `1` se start kar sakta hai. Har baar 1 ya 2 steps le sakta hai. **Minimum cost** se top tak pahunch.

**Example:** `cost = [10, 15, 20]` → Output: `15` (Start at index 1, pay 15, then 2 steps to top)

### 💡 Concept

```
minCost(i) = cost[i] + min(minCost(i-1), minCost(i-2))
Base: minCost(0) = cost[0], minCost(1) = cost[1]
Top = min(minCost(n-1), minCost(n-2)) + special handling
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    public int minCostClimbingStairs(int[] cost) {
        int n = cost.length;
        int[] memo = new int[n];
        Arrays.fill(memo, -1);
        
        // Top can be reached from last step or second last step
        return Math.min(solve(cost, n - 1, memo), solve(cost, n - 2, memo));
    }
    
    private int solve(int[] cost, int i, int[] memo) {
        // Base case: first two steps ka cost hi unka min cost hai
        if (i < 0) return 0;
        if (i == 0 || i == 1) return cost[i];
        
        if (memo[i] != -1) return memo[i];
        
        // Current step ka cost + min of reaching previous two steps
        memo[i] = cost[i] + Math.min(solve(cost, i - 1, memo), solve(cost, i - 2, memo));
        return memo[i];
    }
}
```

### ✅ Time Complexity: O(n) | Space: O(n)

## 3️⃣ House Robber

### 📖 Description 

Houses circularly nahi hain (line mein hain). Adjacent houses mein chori nahi kar sakte. **Maximum loot** kya ho sakta hai?

**Example:** `nums = [1, 2, 3, 1]` → Output: `4` (Loot house 1 and 3: 1 + 3 = 4)

### 💡 Concept

```
rob(i) = max( rob(i-1), nums[i] + rob(i-2) )
Base: rob(0) = nums[0], rob(1) = max(nums[0], nums[1])
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    public int rob(int[] nums) {
        int n = nums.length;
        if (n == 0) return 0;
        if (n == 1) return nums[0];
        
        int[] memo = new int[n];
        Arrays.fill(memo, -1);
        return solve(nums, n - 1, memo);
    }
    
    private int solve(int[] nums, int i, int[] memo) {
        // Base cases
        if (i < 0) return 0;
        if (i == 0) return nums[0];
        if (i == 1) return Math.max(nums[0], nums[1]);
        
        if (memo[i] != -1) return memo[i];
        
        // Option 1: Don't rob current house -> take best from previous
        // Option 2: Rob current house -> take best from i-2 and add current
        memo[i] = Math.max(solve(nums, i - 1, memo), 
                          solve(nums, i - 2, memo) + nums[i]);
        return memo[i];
    }
}
```

### ✅ Time Complexity: O(n) | Space: O(n)

## 4️⃣ House Robber II

### 📖 Description 

Same as House Robber but houses **circular** hain. First aur last house adjacent hain. Dono ko ek saath loot nahi sakte.

**Example:** `nums = [2, 3, 2]` → Output: `3` (Either loot index 1 only)

### 💡 Concept

```
Circular hone ki wajah se do cases:
Case 1: First house loot lo, last house nahi (range: 0 to n-2)
Case 2: Last house loot lo, first house nahi (range: 1 to n-1)

Answer = max(robLinear(0 to n-2), robLinear(1 to n-1))
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    public int rob(int[] nums) {
        int n = nums.length;
        if (n == 0) return 0;
        if (n == 1) return nums[0];
        if (n == 2) return Math.max(nums[0], nums[1]);
        
        // Case 1: Exclude last house (0 to n-2)
        int case1 = robLinear(nums, 0, n - 2);
        
        // Case 2: Exclude first house (1 to n-1)
        int case2 = robLinear(nums, 1, n - 1);
        
        return Math.max(case1, case2);
    }
    
    private int robLinear(int[] nums, int start, int end) {
        int len = end - start + 1;
        int[] memo = new int[len];
        Arrays.fill(memo, -1);
        return solveLinear(nums, start, end, end, memo);
    }
    
    private int solveLinear(int[] nums, int start, int end, int i, int[] memo) {
        int idx = i - start; // Adjust index for memo array
        if (i < start) return 0;
        if (i == start) return nums[start];
        if (i == start + 1) return Math.max(nums[start], nums[start + 1]);
        
        if (memo[idx] != -1) return memo[idx];
        
        memo[idx] = Math.max(solveLinear(nums, start, end, i - 1, memo),
                            solveLinear(nums, start, end, i - 2, memo) + nums[i]);
        return memo[idx];
    }
}
```

### ✅ Time Complexity: O(n) | Space: O(n)

## 5️⃣ Longest Palindromic Substring

### 📖 Description 

String mein se **longest palindrome** substring dhundhna hai. Palindrome wahi jo aage aur piche same read ho.

**Example:** `s = "babad"` → Output: `"bab"` ya `"aba"`

### 💡 Concept

```
Palindrome condition:
1. Single char → always palindrome
2. Two chars → palindrome if both chars same
3. Length > 2 → s[i] == s[j] AND inner substring (i+1 to j-1) is palindrome

DP state: dp[i][j] = true if s[i..j] is palindrome
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    private String s;
    private int[][] memo; // -1: not computed, 0: false, 1: true
    private int maxLen;
    private int startIdx;
    
    public String longestPalindrome(String s) {
        this.s = s;
        int n = s.length();
        memo = new int[n][n];
        for (int i = 0; i < n; i++) {
            Arrays.fill(memo[i], -1);
        }
        
        maxLen = 1;
        startIdx = 0;
        
        // Check all possible substrings
        for (int i = 0; i < n; i++) {
            for (int j = i; j < n; j++) {
                if (isPalindrome(i, j)) {
                    int len = j - i + 1;
                    if (len > maxLen) {
                        maxLen = len;
                        startIdx = i;
                    }
                }
            }
        }
        
        return s.substring(startIdx, startIdx + maxLen);
    }
    
    private boolean isPalindrome(int i, int j) {
        // Base cases
        if (i > j) return false;
        if (i == j) return true; // Single char
        if (i + 1 == j) { // Two chars
            return s.charAt(i) == s.charAt(j);
        }
        
        // If already computed
        if (memo[i][j] != -1) return memo[i][j] == 1;
        
        // Recursive condition
        boolean result = (s.charAt(i) == s.charAt(j)) && isPalindrome(i + 1, j - 1);
        memo[i][j] = result ? 1 : 0;
        return result;
    }
}
```

### ✅ Time Complexity: O(n²) | Space: O(n²)

## 6️⃣ Palindromic Substrings

### 📖 Description 

String mein **total kitne palindromic substrings** hain? Single character bhi count hoga, aur different indices ke same characters alag count honge.

**Example:** `s = "aaa"` → Substrings: "a"(0), "a"(1), "a"(2), "aa"(0-1), "aa"(1-2), "aaa"(0-2) → Output: `6`

### 💡 Concept

```
Same logic as longest palindromic substring, but count all true dp[i][j]
Instead of tracking max, just increment counter.
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    private String s;
    private int[][] memo; // -1: not computed, 0: false, 1: true
    private int count;
    
    public int countSubstrings(String s) {
        this.s = s;
        int n = s.length();
        memo = new int[n][n];
        for (int i = 0; i < n; i++) {
            Arrays.fill(memo[i], -1);
        }
        
        count = 0;
        
        // Check all substrings
        for (int i = 0; i < n; i++) {
            for (int j = i; j < n; j++) {
                if (isPalindrome(i, j)) {
                    count++;
                }
            }
        }
        
        return count;
    }
    
    private boolean isPalindrome(int i, int j) {
        if (i > j) return false;
        if (i == j) return true;
        if (i + 1 == j) {
            return s.charAt(i) == s.charAt(j);
        }
        
        if (memo[i][j] != -1) return memo[i][j] == 1;
        
        boolean result = (s.charAt(i) == s.charAt(j)) && isPalindrome(i + 1, j - 1);
        memo[i][j] = result ? 1 : 0;
        return result;
    }
}
```

### ✅ Time Complexity: O(n²) | Space: O(n²)

## 7️⃣ Decode Ways

### 📖 Description 

Digits string di hai: `'A'=1, 'B'=2, ..., 'Z'=26`. Is string ko decode karne ke **kitne tareeqe** hain?

**Example:** `s = "226"` → Decodings: "BZ"(2,26), "VF"(22,6), "BBF"(2,2,6) → Output: `3`

### 💡 Concept

```
decodings(i) = decodings(i+1) (if s[i] != '0') + decodings(i+2) (if substring(i,i+2) in 10-26)
Base: If i reaches end → 1 way
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    private String s;
    private int[] memo;
    
    public int numDecodings(String s) {
        this.s = s;
        int n = s.length();
        if (n == 0) return 0;
        memo = new int[n];
        Arrays.fill(memo, -1);
        return solve(0);
    }
    
    private int solve(int i) {
        // Base: successfully reached end of string
        if (i == s.length()) return 1;
        
        // Current char '0' → can't start decoding
        if (s.charAt(i) == '0') return 0;
        
        if (memo[i] != -1) return memo[i];
        
        // Case 1: Take single digit (1-9)
        int ways = solve(i + 1);
        
        // Case 2: Take two digits (10-26) if possible
        if (i + 1 < s.length()) {
            int twoDigit = Integer.parseInt(s.substring(i, i + 2));
            if (twoDigit >= 10 && twoDigit <= 26) {
                ways += solve(i + 2);
            }
        }
        
        memo[i] = ways;
        return ways;
    }
}
```

### ✅ Time Complexity: O(n) | Space: O(n)

## 8️⃣ Coin Change (Minimum Coins)

### 📖 Description 

Given coins denominations and an amount. Minimum coins required to make that amount. Infinite supply of each coin.

**Example:** `coins = [1,2,5], amount = 11` → Output: `3` (5+5+1)

### 💡 Concept

```
minCoins(amount) = min over all coins (1 + minCoins(amount - coin))
Base: amount == 0 → 0
      amount < 0 → infinity (invalid)
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    private int[] coins;
    private int[] memo;
    
    public int coinChange(int[] coins, int amount) {
        this.coins = coins;
        memo = new int[amount + 1];
        Arrays.fill(memo, -2); // -2 means not computed
        int result = solve(amount);
        return result == Integer.MAX_VALUE ? -1 : result;
    }
    
    private int solve(int amount) {
        // Base case
        if (amount == 0) return 0;
        if (amount < 0) return Integer.MAX_VALUE;
        
        if (memo[amount] != -2) return memo[amount];
        
        int minCoins = Integer.MAX_VALUE;
        
        // Try all coins
        for (int coin : coins) {
            int subResult = solve(amount - coin);
            if (subResult != Integer.MAX_VALUE) {
                minCoins = Math.min(minCoins, subResult + 1);
            }
        }
        
        memo[amount] = minCoins;
        return minCoins;
    }
}
```

### ✅ Time Complexity: O(amount × coins.length) | Space: O(amount)

## 9️⃣ Maximum Product Subarray

### 📖 Description 

Array mein contiguous subarray ka **maximum product** find karna hai.

**Example:** `nums = [2,3,-2,4]` → Output: `6` (subarray [2,3] product 6)

### 💡 Concept

```
Yahan tricky part negative numbers hai.
Negative × Negative = Positive
Isliye hum track karte hain:
- maxEndingHere: maximum product ending at i
- minEndingHere: minimum product ending at i (most negative)
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    private int[] nums;
    private int[][] memo; // memo[i][0] = max, memo[i][1] = min
    
    public int maxProduct(int[] nums) {
        this.nums = nums;
        int n = nums.length;
        memo = new int[n][2];
        for (int i = 0; i < n; i++) {
            Arrays.fill(memo[i], Integer.MIN_VALUE);
        }
        
        solve(n - 1);
        
        // Find maximum from all positions
        int maxProduct = nums[0];
        for (int i = 0; i < n; i++) {
            maxProduct = Math.max(maxProduct, memo[i][0]);
        }
        return maxProduct;
    }
    
    private void solve(int i) {
        if (i == 0) {
            memo[0][0] = nums[0]; // max
            memo[0][1] = nums[0]; // min
            return;
        }
        
        solve(i - 1);
        
        int current = nums[i];
        
        // Three possibilities for max:
        // 1. Current alone
        // 2. Current × previous max
        // 3. Current × previous min (if negative × negative)
        int maxVal = Math.max(current, 
                       Math.max(memo[i - 1][0] * current, 
                                memo[i - 1][1] * current));
        
        int minVal = Math.min(current,
                       Math.min(memo[i - 1][0] * current,
                                memo[i - 1][1] * current));
        
        memo[i][0] = maxVal;
        memo[i][1] = minVal;
    }
}
```

### ✅ Time Complexity: O(n) | Space: O(n)

## 🔟 Word Break

### 📖 Description 

String `s` ko dictionary words mein break kar sakte hain ya nahi? Word dictionary se multiple baar use ho sakta hai.

**Example:** `s = "leetcode", wordDict = ["leet","code"]` → Output: `true`

### 💡 Concept

```
wordBreak(i) = true if substring i to end can be segmented
Check: s[i..k] in dictionary AND wordBreak(k+1) is true
```

###  Recursion + Memoization Solution

```java
import java.util.*;

class Solution {
    private String s;
    private Set<String> wordSet;
    private int[] memo; // -1: not computed, 0: false, 1: true
    
    public boolean wordBreak(String s, List<String> wordDict) {
        this.s = s;
        this.wordSet = new HashSet<>(wordDict);
        memo = new int[s.length()];
        Arrays.fill(memo, -1);
        return solve(0);
    }
    
    private boolean solve(int start) {
        // Reached end of string successfully
        if (start == s.length()) return true;
        
        if (memo[start] != -1) return memo[start] == 1;
        
        // Try all possible end positions
        for (int end = start + 1; end <= s.length(); end++) {
            String word = s.substring(start, end);
            if (wordSet.contains(word) && solve(end)) {
                memo[start] = 1;
                return true;
            }
        }
        
        memo[start] = 0;
        return false;
    }
}
```

### ✅ Time Complexity: O(n²) worst case | Space: O(n)

## 1️⃣1️⃣ Longest Increasing Subsequence

### 📖 Description 

Array mein **longest increasing subsequence** (not necessarily contiguous) ki length find karni hai.

**Example:** `nums = [10,9,2,5,3,7,101,18]` → Output: `4` ([2,3,7,101] or [2,3,7,18])

### 💡 Concept

```
LIS(i) = 1 + max(LIS(j)) where j < i and nums[j] < nums[i]
Base: LIS(i) = 1 (single element)
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    private int[] nums;
    private int[] memo;
    
    public int lengthOfLIS(int[] nums) {
        this.nums = nums;
        int n = nums.length;
        memo = new int[n];
        Arrays.fill(memo, -1);
        
        int maxLen = 0;
        // Try each element as starting/ending point
        for (int i = 0; i < n; i++) {
            maxLen = Math.max(maxLen, solve(i));
        }
        return maxLen;
    }
    
    private int solve(int i) {
        if (memo[i] != -1) return memo[i];
        
        int maxLen = 1; // At least the element itself
        
        // Check all previous elements
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                maxLen = Math.max(maxLen, solve(j) + 1);
            }
        }
        
        memo[i] = maxLen;
        return maxLen;
    }
}
```

### ✅ Time Complexity: O(n²) | Space: O(n)

## 1️⃣2️⃣ Partition Equal Subset Sum

### 📖 Description 

Array ko do subsets mein divide karna hai jinka sum equal ho. Possible hai ya nahi?

**Example:** `nums = [1,5,11,5]` → Output: `true` ([1,5,5] and [11], both sum=11)

### 💡 Concept

```
Target = totalSum / 2
Problem reduces to: Does there exist a subset with sum = target?
SubsetSum(i, sum) = SubsetSum(i-1, sum) OR SubsetSum(i-1, sum - nums[i])
```

###  Recursion + Memoization Solution

```java
import java.util.Arrays;

class Solution {
    private int[] nums;
    private int[][] memo; // -1: not computed, 0: false, 1: true
    private int target;
    
    public boolean canPartition(int[] nums) {
        int total = 0;
        for (int num : nums) total += num;
        
        // If total sum is odd, can't partition equally
        if (total % 2 != 0) return false;
        
        this.target = total / 2;
        this.nums = nums;
        memo = new int[nums.length][target + 1];
        for (int i = 0; i < nums.length; i++) {
            Arrays.fill(memo[i], -1);
        }
        
        return subsetSum(0, 0);
    }
    
    private boolean subsetSum(int idx, int currentSum) {
        // Found a subset that sums to target
        if (currentSum == target) return true;
        
        // Exceeded target or ran out of elements
        if (currentSum > target || idx >= nums.length) return false;
        
        if (memo[idx][currentSum] != -1) return memo[idx][currentSum] == 1;
        
        // Option 1: Exclude current element
        // Option 2: Include current element
        boolean result = subsetSum(idx + 1, currentSum) || 
                         subsetSum(idx + 1, currentSum + nums[idx]);
        
        memo[idx][currentSum] = result ? 1 : 0;
        return result;
    }
}
```

### ✅ Time Complexity: O(n × target) | Space: O(n × target)

## 🎯 Summary Table

| Problem | Recursive Relation | Base Cases | Time |
|---------|-------------------|------------|------|
| Climbing Stairs | `f(n) = f(n-1) + f(n-2)` | f(1)=1, f(2)=2 | O(n) |
| Min Cost Climbing | `f(i) = cost[i] + min(f(i-1), f(i-2))` | f(0)=cost[0], f(1)=cost[1] | O(n) |
| House Robber | `f(i) = max(f(i-1), nums[i] + f(i-2))` | f(0)=nums[0], f(1)=max(nums[0],nums[1]) | O(n) |
| House Robber II | `max(rob(0,n-2), rob(1,n-1))` | Same as House Robber | O(n) |
| Longest Pal Substring | `dp[i][j] = s[i]==s[j] && dp[i+1][j-1]` | Single char = true | O(n²) |
| Palindromic Substrings | Same as above, count all true | Same as above | O(n²) |
| Decode Ways | `f(i) = f(i+1) + f(i+2)` if valid | f(n)=1 | O(n) |
| Coin Change | `f(a) = 1 + min(f(a-coin))` for all coins | f(0)=0 | O(a×c) |
| Max Product Subarray | Track max and min | max=min=first element | O(n) |
| Word Break | `f(i) = OR of f(j) for words matching` | f(n)=true | O(n²) |
| LIS | `f(i) = 1 + max(f(j)) where j<i and nums[j]<nums[i]` | f(i)=1 | O(n²) |
| Partition Equal | `f(i,sum) = f(i+1,sum) OR f(i+1,sum+nums[i])` | sum==target = true | O(n×target) |

## 💪 Pro Tips for DP Interviews

1. **Identify DP**: Jab problem mein "maximum/minimum", "number of ways", "possible or not" ho aur overlapping subproblems ho → DP

2. **State Definition**: Soch ki mujhe kya track karna hai? (index, sum, etc.)

3. **Draw Recursion Tree**: Paper par kuch examples bana ke dekho, pattern dikhega

4. **Memoization vs Tabulation**:
   - Memoization: Easy to implement from recursion, top-down
   - Tabulation: More efficient, bottom-up
