

# DP on Trees Pattern

## Tree DP ke 10 Main Variants:

| Variant | Description | Approach |
|---------|-------------|----------|
| **Diameter of Tree** | Longest path between any two nodes | Max depth from each node |
| **Tree Height** | Height of tree | 1 + max(child heights) |
| **Max Path Sum** | Maximum sum path (any node to any node) | Track max path through node |
| **House Robber III** | Max sum with no adjacent nodes | DP[node] = {take, skip} |
| **Binary Tree Cameras** | Min cameras to cover all nodes | States: covered/has camera/needs cover |
| **Tree Coloring** | Color tree with constraints | DP with colors |
| **Tree Distance** | Sum of distances between all nodes | Re-rooting DP |
| **Tree Partition** | Split tree into components | DP with size constraint |
| **Tree Matching** | Maximum matching in tree | Take or skip edge |
| **LCA with DP** | Lowest Common Ancestor with preprocessing | Binary lifting |

## 1. **Diameter of Binary Tree (Leetcode 543)**

**Instinct:**  
Tree ka longest path between any two nodes. Har node ke liye, left subtree ka height + right subtree ka height = path through that node. Max of all such paths.

**Approach:**  
1. DFS se har node ka height calculate karo
2. Har node pe diameter = leftHeight + rightHeight
3. Global max track karo

```java
class Solution {
    int maxDiameter = 0;
    
    public int diameterOfBinaryTree(TreeNode root) {
        dfs(root);
        return maxDiameter;
    }
    
    private int dfs(TreeNode node) {
        if (node == null) return 0;
        
        int leftHeight = dfs(node.left);
        int rightHeight = dfs(node.right);
        
        // path through current node
        maxDiameter = Math.max(maxDiameter, leftHeight + rightHeight);
        
        // return height
        return 1 + Math.max(leftHeight, rightHeight);
    }
}
```

**Key Insight:** Diameter = max(leftHeight + rightHeight) for all nodes. Height = 1 + max(left, right).

## 2. **Binary Tree Maximum Path Sum (Leetcode 124)**

**Instinct:**  
Maximum sum path (any node to any node). Har node ke liye, max sum path through that node = node.val + max(0, leftMax) + max(0, rightMax).

**Approach:**  
1. DFS se har node ka max sum path starting from that node (going down only)
2. Path through node = node.val + max(0, left) + max(0, right)
3. Global max track karo

```java
class Solution {
    int maxSum = Integer.MIN_VALUE;
    
    public int maxPathSum(TreeNode root) {
        dfs(root);
        return maxSum;
    }
    
    private int dfs(TreeNode node) {
        if (node == null) return 0;
        
        int left = Math.max(0, dfs(node.left));
        int right = Math.max(0, dfs(node.right));
        
        // path through current node
        maxSum = Math.max(maxSum, node.val + left + right);
        
        // return max sum path starting from node (going down)
        return node.val + Math.max(left, right);
    }
}
```

**Key Insight:** Negative paths ko ignore karo (max with 0). Path through node = left + right + node.val.

## 3. **House Robber III (Leetcode 337)**

**Instinct:**  
Tree me rob karna hai, adjacent nodes cannot be robbed. Har node ke liye do options: rob current node (then skip children) ya skip current node (then take best from children).

**Approach:**  
1. DFS returns {robCurrent, skipCurrent}
2. robCurrent = node.val + left.skip + right.skip
3. skipCurrent = max(left.rob, left.skip) + max(right.rob, right.skip)

```java
class Solution {
    public int rob(TreeNode root) {
        int[] res = dfs(root);
        return Math.max(res[0], res[1]);
    }
    
    private int[] dfs(TreeNode node) {
        if (node == null) return new int[]{0, 0};
        
        int[] left = dfs(node.left);
        int[] right = dfs(node.right);
        
        int rob = node.val + left[1] + right[1];
        int skip = Math.max(left[0], left[1]) + Math.max(right[0], right[1]);
        
        return new int[]{rob, skip};
    }
}
```

**Key Insight:** Return pair (rob, skip). Children ke rob/skip ko combine karo.

## 4. **Binary Tree Cameras (Leetcode 968)**

**Instinct:**  
Tree me minimum cameras lagaani hain such that every node is covered. Har node ke 3 states hain: has camera, covered by child, needs cover.

**Approach:**  
1. DFS returns state: 0 = needs cover, 1 = has camera, 2 = covered
2. If any child needs cover, current must have camera
3. If any child has camera, current is covered
4. Else current needs cover

```java
class Solution {
    int cameras = 0;
    
    public int minCameraCover(TreeNode root) {
        int state = dfs(root);
        // if root needs cover, add camera
        return cameras + (state == 0 ? 1 : 0);
    }
    
    // 0 = needs cover, 1 = has camera, 2 = covered
    private int dfs(TreeNode node) {
        if (node == null) return 2;
        
        int left = dfs(node.left);
        int right = dfs(node.right);
        
        // if any child needs cover, put camera here
        if (left == 0 || right == 0) {
            cameras++;
            return 1;
        }
        
        // if any child has camera, current is covered
        if (left == 1 || right == 1) {
            return 2;
        }
        
        // else current needs cover
        return 0;
    }
}
```

**Key Insight:** Three states - needs cover, has camera, covered. Greedy from leaves.

## 6. **Sum of Distances in Tree (Leetcode 834)**

**Instinct:**  
Har node se baaki saare nodes ke distance ka sum nikaalna hai. Re-rooting DP technique use karo - pehle root 0 ke liye calculate karo, phir parent se child me shift karte time update karo.

**Approach:**  
1. First DFS: count of nodes in each subtree, sum of distances from root
2. Second DFS (re-rooting): when moving from parent to child, formula updates

```java
class Solution {
    int[] ans;
    int[] count;
    List<Integer>[] graph;
    int n;
    
    public int[] sumOfDistancesInTree(int n, int[][] edges) {
        this.n = n;
        graph = new List[n];
        for (int i = 0; i < n; i++) graph[i] = new ArrayList<>();
        for (int[] e : edges) {
            graph[e[0]].add(e[1]);
            graph[e[1]].add(e[0]);
        }
        
        ans = new int[n];
        count = new int[n];
        
        dfs1(0, -1);
        dfs2(0, -1);
        
        return ans;
    }
    
    private void dfs1(int node, int parent) {
        count[node] = 1;
        for (int child : graph[node]) {
            if (child == parent) continue;
            dfs1(child, node);
            count[node] += count[child];
            ans[node] += ans[child] + count[child];
        }
    }
    
    private void dfs2(int node, int parent) {
        for (int child : graph[node]) {
            if (child == parent) continue;
            // re-root from node to child
            ans[child] = ans[node] - count[child] + (n - count[child]);
            dfs2(child, node);
        }
    }
}
```

**Key Insight:** Re-rooting formula: ans[child] = ans[parent] - count[child] + (n - count[child]).

## 7. **Longest Path With Different Adjacent Characters (Leetcode 2246)**

**Instinct:**  
Tree me longest path where adjacent nodes have different characters. Har node ke liye, top 2 longest paths from children with different character.

**Approach:**  
1. DFS returns longest path starting from node (going down)
2. Track top 2 longest valid child paths
3. Path through node = sum of top 2 + 1

```java
class Solution {
    int maxLen = 0;
    String s;
    List<Integer>[] graph;
    
    public int longestPath(int[] parent, String s) {
        this.s = s;
        int n = parent.length;
        graph = new List[n];
        for (int i = 0; i < n; i++) graph[i] = new ArrayList<>();
        for (int i = 1; i < n; i++) {
            graph[parent[i]].add(i);
        }
        
        dfs(0);
        return maxLen;
    }
    
    private int dfs(int node) {
        int top1 = 0, top2 = 0;
        
        for (int child : graph[node]) {
            int childLen = dfs(child);
            if (s.charAt(child) != s.charAt(node)) {
                if (childLen > top1) {
                    top2 = top1;
                    top1 = childLen;
                } else if (childLen > top2) {
                    top2 = childLen;
                }
            }
        }
        
        maxLen = Math.max(maxLen, 1 + top1 + top2);
        return 1 + top1;
    }
}
```

**Key Insight:** Only consider children with different character. Path = 1 + top1 + top2.

## 8. **Minimum Time to Collect All Apples in Tree (Leetcode 1443)**

**Instinct:**  
Tree me apples hain. Root se start karo, saare apples collect karke wapas aana hai. Minimum time = 2 * (edges needed to reach all apples).

**Approach:**  
1. DFS returns whether subtree has any apple
2. If child has apple, add 2 to time (going and coming back)

```java
class Solution {
    List<Integer>[] graph;
    List<Boolean> hasApple;
    
    public int minTime(int n, int[][] edges, List<Boolean> hasApple) {
        this.hasApple = hasApple;
        graph = new List[n];
        for (int i = 0; i < n; i++) graph[i] = new ArrayList<>();
        for (int[] e : edges) {
            graph[e[0]].add(e[1]);
            graph[e[1]].add(e[0]);
        }
        
        return dfs(0, -1);
    }
    
    private int dfs(int node, int parent) {
        int time = 0;
        for (int child : graph[node]) {
            if (child == parent) continue;
            int childTime = dfs(child, node);
            if (childTime > 0 || hasApple.get(child)) {
                time += 2 + childTime;
            }
        }
        return time;
    }
}
```

**Key Insight:** Only traverse edges that lead to apples. Each edge takes 2 time (go and return).

## 9. **Tree Coloring (3-color tree with no same color adjacent)**

**Instinct:**  
Tree ko 3 colors me color karna hai such that no two adjacent nodes have same color. Count number of ways.

**Approach:**  
1. DP[node][color] = ways to color subtree with node having given color
2. For each child, sum over all colors except current

```java
class Solution {
    int MOD = 1_000_000_007;
    List<Integer>[] graph;
    long[][] dp;
    
    public int colorTree(int n, int[][] edges) {
        graph = new List[n];
        for (int i = 0; i < n; i++) graph[i] = new ArrayList<>();
        for (int[] e : edges) {
            graph[e[0]].add(e[1]);
            graph[e[1]].add(e[0]);
        }
        
        dp = new long[n][3];
        for (int i = 0; i < n; i++) Arrays.fill(dp[i], -1);
        
        long total = 0;
        for (int c = 0; c < 3; c++) {
            total = (total + dfs(0, -1, c)) % MOD;
        }
        return (int) total;
    }
    
    private long dfs(int node, int parent, int color) {
        if (dp[node][color] != -1) return dp[node][color];
        
        long ways = 1;
        for (int child : graph[node]) {
            if (child == parent) continue;
            long childWays = 0;
            for (int c = 0; c < 3; c++) {
                if (c == color) continue;
                childWays = (childWays + dfs(child, node, c)) % MOD;
            }
            ways = (ways * childWays) % MOD;
        }
        
        dp[node][color] = ways;
        return ways;
    }
}
```

**Key Insight:** Multiplication over children. Child cannot have same color as parent.

## 10. **Maximum Product of Splitted Binary Tree (Leetcode 1339)**

**Instinct:**  
Binary tree ko ek edge remove karke do subtrees me split karna hai. Product of sums maximize karna hai.

**Approach:**  
1. First DFS: calculate total sum
2. Second DFS: for each edge, calculate subtree sum, product = sum * (total - sum)

```java
class Solution {
    long total = 0;
    long maxProduct = 0;
    int MOD = 1_000_000_007;
    
    public int maxProduct(TreeNode root) {
        total = getSum(root);
        dfs(root);
        return (int)(maxProduct % MOD);
    }
    
    private long getSum(TreeNode node) {
        if (node == null) return 0;
        return node.val + getSum(node.left) + getSum(node.right);
    }
    
    private long dfs(TreeNode node) {
        if (node == null) return 0;
        
        long leftSum = dfs(node.left);
        long rightSum = dfs(node.right);
        long subtreeSum = node.val + leftSum + rightSum;
        
        if (subtreeSum != total) {
            maxProduct = Math.max(maxProduct, subtreeSum * (total - subtreeSum));
        }
        
        return subtreeSum;
    }
}
```

**Key Insight:** Remove edge between node and parent. Product = subtreeSum * (total - subtreeSum).

## 11. **Minimum Number of Operations to Make Tree Connected (Leetcode 1319)**

**Instinct:**  
Graph (tree-like) me minimum edges add karni hain to make connected. Components count - 1.

**Approach:**  
1. Count number of connected components using DFS
2. Answer = components - 1

```java
class Solution {
    List<Integer>[] graph;
    boolean[] visited;
    
    public int makeConnected(int n, int[][] connections) {
        if (connections.length < n - 1) return -1;
        
        graph = new List[n];
        for (int i = 0; i < n; i++) graph[i] = new ArrayList<>();
        for (int[] c : connections) {
            graph[c[0]].add(c[1]);
            graph[c[1]].add(c[0]);
        }
        
        visited = new boolean[n];
        int components = 0;
        for (int i = 0; i < n; i++) {
            if (!visited[i]) {
                dfs(i);
                components++;
            }
        }
        
        return components - 1;
    }
    
    private void dfs(int node) {
        visited[node] = true;
        for (int neighbor : graph[node]) {
            if (!visited[neighbor]) {
                dfs(neighbor);
            }
        }
    }
}
```

**Key Insight:** Connected components count. Need components-1 edges to connect them.

## 12. **Minimum Edge Reversals to Make Root (Leetcode 1766)**

**Instinct:**  
Directed tree me root change karne ke liye minimum edge reversals. Re-rooting DP.

**Approach:**  
1. First DFS: count reversals needed for root 0
2. Second DFS: when moving from parent to child, update count

```java
class Solution {
    List<int[]>[] graph;
    int[] ans;
    
    public int[] minEdgeReversals(int n, int[][] edges) {
        graph = new List[n];
        for (int i = 0; i < n; i++) graph[i] = new ArrayList<>();
        for (int[] e : edges) {
            graph[e[0]].add(new int[]{e[1], 1});      // forward edge
            graph[e[1]].add(new int[]{e[0], -1});     // backward edge (needs reversal)
        }
        
        ans = new int[n];
        dfs1(0, -1);
        dfs2(0, -1, 0);
        
        return ans;
    }
    
    private void dfs1(int node, int parent) {
        for (int[] next : graph[node]) {
            int child = next[0];
            int dir = next[1];
            if (child == parent) continue;
            if (dir == -1) ans[0]++; // edge needs reversal
            dfs1(child, node);
        }
    }
    
    private void dfs2(int node, int parent, int parentReversals) {
        ans[node] = parentReversals;
        for (int[] next : graph[node]) {
            int child = next[0];
            int dir = next[1];
            if (child == parent) continue;
            
            int newReversals = parentReversals;
            if (dir == 1) {
                newReversals++; // going from parent to child, edge is forward
            } else {
                newReversals--; // going from parent to child, edge is backward
            }
            dfs2(child, node, newReversals);
        }
    }
}
```

**Key Insight:** Track edge direction. When re-rooting, reverse the edge direction cost.

## 13. **Tree Diameter with DP (Generic Tree)**

**Instinct:**  
Generic tree (not binary) ka diameter. Same logic: max of (top two longest paths from children).

**Approach:**  
1. DFS returns max depth
2. Track top two longest child depths

```java
class Solution {
    int diameter = 0;
    List<Integer>[] graph;
    
    public int treeDiameter(int[][] edges) {
        int n = edges.length + 1;
        graph = new List[n];
        for (int i = 0; i < n; i++) graph[i] = new ArrayList<>();
        for (int[] e : edges) {
            graph[e[0]].add(e[1]);
            graph[e[1]].add(e[0]);
        }
        
        dfs(0, -1);
        return diameter;
    }
    
    private int dfs(int node, int parent) {
        int top1 = 0, top2 = 0;
        
        for (int child : graph[node]) {
            if (child == parent) continue;
            int depth = dfs(child, node);
            if (depth > top1) {
                top2 = top1;
                top1 = depth;
            } else if (depth > top2) {
                top2 = depth;
            }
        }
        
        diameter = Math.max(diameter, top1 + top2);
        return 1 + top1;
    }
}
```

**Key Insight:** Same as binary tree diameter. Track top two longest child paths.

## 14. **Maximum Path Sum Between Two Leaves (Binary Tree)**

**Instinct:**  
Two leaves ke beech maximum sum path. Leaf to leaf path. Similar to diameter but with values.

**Approach:**  
1. DFS returns max path sum from node to any leaf
2. Update global max with left + right + node.val (if both children exist)

```java
class Solution {
    int maxSum = Integer.MIN_VALUE;
    
    public int maxPathSumBetweenLeaves(TreeNode root) {
        dfs(root);
        return maxSum;
    }
    
    private int dfs(TreeNode node) {
        if (node == null) return Integer.MIN_VALUE / 2;
        
        if (node.left == null && node.right == null) {
            return node.val;
        }
        
        int left = dfs(node.left);
        int right = dfs(node.right);
        
        if (node.left != null && node.right != null) {
            maxSum = Math.max(maxSum, left + right + node.val);
        }
        
        int maxChild = Math.max(left, right);
        return node.val + maxChild;
    }
}
```

**Key Insight:** Path must go through node with both children. Leaf-to-leaf path.

## 15. **Largest BST Subtree (Leetcode 333)**

**Instinct:**  
Binary tree me largest subtree which is a BST. Har node ke liye check karo: min, max, size, isBST.

**Approach:**  
1. DFS returns {min, max, size, isBST}
2. If left.isBST && right.isBST && left.max < node.val < right.min → valid BST

```java
class Solution {
    int maxSize = 0;
    
    public int largestBSTSubtree(TreeNode root) {
        dfs(root);
        return maxSize;
    }
    
    private int[] dfs(TreeNode node) {
        // return {min, max, size, isBST}
        if (node == null) return new int[]{Integer.MAX_VALUE, Integer.MIN_VALUE, 0, 1};
        
        int[] left = dfs(node.left);
        int[] right = dfs(node.right);
        
        if (left[3] == 1 && right[3] == 1 && 
            node.val > left[1] && node.val < right[0]) {
            int size = 1 + left[2] + right[2];
            maxSize = Math.max(maxSize, size);
            return new int[]{
                Math.min(node.val, left[0]),
                Math.max(node.val, right[1]),
                size, 1
            };
        }
        
        return new int[]{0, 0, Math.max(left[2], right[2]), 0};
    }
}
```

**Key Insight:** Track min, max, size, isBST. Validate BST property.

## 16. **Tree Pruning (Remove subtrees with sum zero)**

**Instinct:**  
Tree me se aise subtrees remove karo jinka sum 0 ho. Post-order DFS.

**Approach:**  
1. DFS returns sum of subtree
2. If sum == 0, return 0 (prune)

```java
class Solution {
    public TreeNode pruneTree(TreeNode root) {
        if (dfs(root) == 0) return null;
        return root;
    }
    
    private int dfs(TreeNode node) {
        if (node == null) return 0;
        
        int leftSum = dfs(node.left);
        int rightSum = dfs(node.right);
        
        if (leftSum == 0) node.left = null;
        if (rightSum == 0) node.right = null;
        
        return node.val + leftSum + rightSum;
    }
}
```

**Key Insight:** Post-order traversal. If subtree sum = 0, prune it.

## 17. **Binary Tree Paths with Sum (Leetcode 437)**

**Instinct:**  
Path sum equal to target (path can start/end anywhere). Prefix sum technique on tree.

**Approach:**  
1. DFS with map storing prefix sums
2. For current node, check if (currentSum - target) exists in map

```java
class Solution {
    int count = 0;
    Map<Integer, Integer> prefixMap = new HashMap<>();
    
    public int pathSum(TreeNode root, int targetSum) {
        prefixMap.put(0, 1);
        dfs(root, 0, targetSum);
        return count;
    }
    
    private void dfs(TreeNode node, int currSum, int target) {
        if (node == null) return;
        
        currSum += node.val;
        count += prefixMap.getOrDefault(currSum - target, 0);
        
        prefixMap.put(currSum, prefixMap.getOrDefault(currSum, 0) + 1);
        
        dfs(node.left, currSum, target);
        dfs(node.right, currSum, target);
        
        prefixMap.put(currSum, prefixMap.get(currSum) - 1);
    }
}
```

**Key Insight:** Prefix sum technique on tree. Backtrack when returning.

## 18. **Minimum Distance Between BST Nodes (Leetcode 783)**

**Instinct:**  
BST me minimum difference between any two nodes. In-order traversal gives sorted order.

**Approach:**  
1. In-order traversal
2. Track previous node value
3. Update min difference

```java
class Solution {
    int minDiff = Integer.MAX_VALUE;
    Integer prev = null;
    
    public int minDiffInBST(TreeNode root) {
        dfs(root);
        return minDiff;
    }
    
    private void dfs(TreeNode node) {
        if (node == null) return;
        
        dfs(node.left);
        
        if (prev != null) {
            minDiff = Math.min(minDiff, node.val - prev);
        }
        prev = node.val;
        
        dfs(node.right);
    }
}
```

**Key Insight:** In-order traversal of BST gives sorted order. Min difference between adjacent.

## 19. **Recover Binary Search Tree (Leetcode 99)**

**Instinct:**  
BST me do nodes swap ho gaye hain. Recover karna hai. In-order traversal se find misplaced nodes.

**Approach:**  
1. In-order traversal to find first and second misplaced nodes
2. Swap their values

```java
class Solution {
    TreeNode first = null;
    TreeNode second = null;
    TreeNode prev = null;
    
    public void recoverTree(TreeNode root) {
        dfs(root);
        int temp = first.val;
        first.val = second.val;
        second.val = temp;
    }
    
    private void dfs(TreeNode node) {
        if (node == null) return;
        
        dfs(node.left);
        
        if (prev != null && prev.val > node.val) {
            if (first == null) {
                first = prev;
            }
            second = node;
        }
        prev = node;
        
        dfs(node.right);
    }
}
```

**Key Insight:** In-order traversal. First violation: prev > current. Second violation: last such occurrence.

## 20. **Maximum Sum BST in Binary Tree (Leetcode 1373)**

**Instinct:**  
Binary tree me largest sum BST subtree. Har node ke liye track karo: min, max, sum, isBST.

**Approach:**  
1. DFS returns {min, max, sum, isBST}
2. If BST, update maxSum
3. If not BST, propagate sum from larger child

```java
class Solution {
    int maxSum = 0;
    
    public int maxSumBST(TreeNode root) {
        dfs(root);
        return maxSum;
    }
    
    private int[] dfs(TreeNode node) {
        // return {min, max, sum, isBST}
        if (node == null) return new int[]{Integer.MAX_VALUE, Integer.MIN_VALUE, 0, 1};
        
        int[] left = dfs(node.left);
        int[] right = dfs(node.right);
        
        if (left[3] == 1 && right[3] == 1 && 
            node.val > left[1] && node.val < right[0]) {
            int sum = node.val + left[2] + right[2];
            maxSum = Math.max(maxSum, sum);
            return new int[]{
                Math.min(node.val, left[0]),
                Math.max(node.val, right[1]),
                sum, 1
            };
        }
        
        return new int[]{0, 0, Math.max(left[2], right[2]), 0};
    }
}
```

**Key Insight:** Same as largest BST subtree but with sum instead of size.

## 🎯 Key Takeaways for Tree DP Pattern

### 1. **Core Structure for Tree DP:**
```java
dfs(node) {
    // process children first (post-order)
    left = dfs(node.left);
    right = dfs(node.right);
    
    // combine results
    result = combine(node.val, left, right);
    
    return result;
}
```

### 2. **Common Return Types from Tree DFS:**

| Return Type | Use Case |
|-------------|----------|
| `int` | Height, sum, max path |
| `int[]` | {min, max, sum, isBST} |
| `int[]` | {rob, skip} |
| `int` (state) | 0/1/2 for camera placement |

### 3. **When to use Tree DP:**
- Problem involves tree structure
- Need to aggregate information from children
- Path between nodes
- Constraints on adjacent nodes
- Re-rooting / rerooting DP

### 4. **Common Tree DP Patterns:**

```
                    Tree DP
                         │
        ┌───────────────┼───────────────┐
        │               │               │
    Path DP         Subtree DP       Re-rooting DP
        │               │               │
    ┌───┼───┐       ┌───┼───┐       ┌───┼───┐
    │   │   │       │   │   │       │   │   │
 Diameter  Sum    BST   Cameras   Sum Dist  Edge
 PathSum  Leaves  Rob   Coloring   Reversals  Pruning
```

### 5. **Important Tree DP Techniques:**

| Technique | Description |
|-----------|-------------|
| **Post-order DFS** | Process children before parent |
| **Re-rooting** | Calculate for root, then shift to children |
| **Top 2 children** | For diameter/path problems |
| **State DP** | Track multiple states per node |
| **Prefix sum** | For path sum problems |

## 📝 Key Formulas to Remember

| Operation | Formula |
|-----------|---------|
| Tree Height | `1 + max(leftHeight, rightHeight)` |
| Diameter | `max(leftHeight + rightHeight)` |
| House Robber | `rob = val + leftSkip + rightSkip; skip = max(leftRob, leftSkip) + max(rightRob, rightSkip)` |
| Re-rooting | `ans[child] = ans[parent] - count[child] + (n - count[child])` |
| BST Check | `left.max < node.val < right.min` |
| Path Sum (Prefix) | `count += map[currSum - target]` |
