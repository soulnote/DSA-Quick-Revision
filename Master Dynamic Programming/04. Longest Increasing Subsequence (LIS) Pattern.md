# Knapsack Pattern

## Knapsack ke 5 Main Variants:

| Variant | Description | Decision |
|---------|-------------|----------|
| **0/1 Knapsack** | Each item ek baar le sakte ho ya nahi | Take or Skip |
| **Unbounded Knapsack** | Each item unlimited baar le sakte ho | Take (stay on same index) or Skip |
| **Bounded Knapsack** | Each item ki limited copies hain | 0 to count times le sakte ho |
| **Fractional Knapsack** | Item ko tod sakte ho (Greedy, not DP) | - |
| **Partition / Subset Sum** | Target sum achieve karna hai | Take or Skip |

## 1. **Partition Equal Subset Sum (Leetcode 416)**

**Instinct:**  
Array ko do subsets me divide karna hai with equal sum. Matlab total sum even hona chahiye, aur target = sum/2 achieve karna hai. Ye **0/1 knapsack** hai - har element ya toh lo ya mat lo.

**Approach:**  
1. Total sum calculate karo, agar odd hai toh false return
2. target = sum/2
3. Memo[i][target] = kya index i se target achieve kar sakte ho
4. Base: target == 0 → true; i == n → false
5. Ya toh current element lo (target - nums[i]) ya mat lo

```java
class Solution {
    int[] nums;
    int n;
    Boolean[][] memo;
    
    public boolean canPartition(int[] nums) {
        this.nums = nums;
        n = nums.length;
        int sum = 0;
        for (int num : nums) sum += num;
        if (sum % 2 != 0) return false;
        
        int target = sum / 2;
        memo = new Boolean[n][target + 1];
        return dfs(0, target);
    }
    
    private boolean dfs(int i, int target) {
        if (target == 0) return true;
        if (i >= n || target < 0) return false;
        if (memo[i][target] != null) return memo[i][target];
        
        // Take current or skip
        boolean take = dfs(i + 1, target - nums[i]);
        boolean skip = dfs(i + 1, target);
        
        memo[i][target] = take || skip;
        return memo[i][target];
    }
}
```

**Key Insight:** Classic 0/1 knapsack - target sum achieve karna hai exactly.

## 2. **Last Stone Weight II (Leetcode 1049)**

**Instinct:**  
Stones ko do groups me divide karna hai. Last stone weight = difference of sums of two groups. Minimum possible difference chahiye. Ye **minimum subset sum difference** problem hai - target = sum/2 ke nearest sum find karo.

**Approach:**  
1. Total sum calculate karo
2. target = sum/2
3. Memo[i][target] = max sum achievable from i with capacity target
4. Answer = sum - 2 * best

```java
class Solution {
    int[] stones;
    int n;
    Integer[][] memo;
    
    public int lastStoneWeightII(int[] stones) {
        this.stones = stones;
        n = stones.length;
        int sum = 0;
        for (int s : stones) sum += s;
        
        int target = sum / 2;
        memo = new Integer[n][target + 1];
        int best = dfs(0, target);
        return sum - 2 * best;
    }
    
    private int dfs(int i, int capacity) {
        if (i >= n || capacity <= 0) return 0;
        if (memo[i][capacity] != null) return memo[i][capacity];
        
        int skip = dfs(i + 1, capacity);
        int take = 0;
        if (stones[i] <= capacity) {
            take = stones[i] + dfs(i + 1, capacity - stones[i]);
        }
        
        memo[i][capacity] = Math.max(skip, take);
        return memo[i][capacity];
    }
}
```

**Key Insight:** Minimum difference = sum - 2 * (max subset sum <= sum/2). 0/1 knapsack.

## 3. **Target Sum (Leetcode 494)**

**Instinct:**  
Har element ke aage + ya - laga kar target sum banana hai. Ye actually **partition with given difference** hai. Let P = positive sum, N = negative sum. P - N = target aur P + N = sum. So P = (target + sum)/2. Problem becomes: kitne subsets hai jinse sum = (target+sum)/2 aata hai.

**Approach:**  
1. total sum calculate karo
2. Agar (target + sum) odd hai ya target > sum, return 0
3. targetSum = (target + sum) / 2
4. Memo[i][target] = ways to achieve target from i

```java
class Solution {
    int[] nums;
    int n;
    Integer[][] memo;
    
    public int findTargetSumWays(int[] nums, int target) {
        this.nums = nums;
        n = nums.length;
        int sum = 0;
        for (int num : nums) sum += num;
        
        if (target > sum || (target + sum) % 2 != 0) return 0;
        int targetSum = (target + sum) / 2;
        
        memo = new Integer[n][targetSum + 1];
        return dfs(0, targetSum);
    }
    
    private int dfs(int i, int target) {
        if (i == n) return target == 0 ? 1 : 0;
        if (target < 0) return 0;
        if (memo[i][target] != null) return memo[i][target];
        
        int take = dfs(i + 1, target - nums[i]);
        int skip = dfs(i + 1, target);
        
        memo[i][target] = take + skip;
        return memo[i][target];
    }
}
```

**Key Insight:** Convert +- problem to subset sum. Ye **count of subsets with given sum** hai.

## 4. **Ones and Zeroes (Leetcode 474)**

**Instinct:**  
Binary strings di hain, m zeros aur n ones use kar sakte ho. Maximum strings select karni hain. Ye **2D 0/1 knapsack** hai - do constraints hain: zeros limit aur ones limit.

**Approach:**  
1. Har string ke zeros aur ones count karo
2. Memo[i][m][n] = max strings from i with m zeros and n ones left
3. Ya toh current string lo (agar fit hoti hai) ya skip karo

```java
class Solution {
    String[] strs;
    int[][][] memo;
    
    public int findMaxForm(String[] strs, int m, int n) {
        this.strs = strs;
        memo = new int[strs.length][m + 1][n + 1];
        for (int i = 0; i < strs.length; i++)
            for (int j = 0; j <= m; j++)
                Arrays.fill(memo[i][j], -1);
        
        return dfs(0, m, n);
    }
    
    private int dfs(int i, int m, int n) {
        if (i == strs.length) return 0;
        if (memo[i][m][n] != -1) return memo[i][m][n];
        
        // count zeros and ones in current string
        int zeros = 0, ones = 0;
        for (char c : strs[i].toCharArray()) {
            if (c == '0') zeros++;
            else ones++;
        }
        
        int skip = dfs(i + 1, m, n);
        int take = 0;
        if (zeros <= m && ones <= n) {
            take = 1 + dfs(i + 1, m - zeros, n - ones);
        }
        
        memo[i][m][n] = Math.max(skip, take);
        return memo[i][m][n];
    }
}
```

**Key Insight:** 2D constraints wala 0/1 knapsack. 3D memoization chahiye.

## 5. **Coin Change (Leetcode 322)**

**Instinct:**  
Amount banane ke liye minimum coins chahiye. Coins unlimited hai - ye **unbounded knapsack** hai. Har coin ko unlimited baar le sakte ho.

**Approach:**  
1. Memo[i][amount] = min coins from index i to make amount
2. Base: amount == 0 → 0; i == n → INF
3. Ya toh current coin lo (i pe hi raho, amount - coins[i]) ya skip karo (i+1)

```java
class Solution {
    int[] coins;
    int n;
    Integer[][] memo;
    int INF = Integer.MAX_VALUE / 2;
    
    public int coinChange(int[] coins, int amount) {
        this.coins = coins;
        n = coins.length;
        memo = new Integer[n][amount + 1];
        int ans = dfs(0, amount);
        return ans >= INF ? -1 : ans;
    }
    
    private int dfs(int i, int amount) {
        if (amount == 0) return 0;
        if (i == n) return INF;
        if (memo[i][amount] != null) return memo[i][amount];
        
        int skip = dfs(i + 1, amount);
        int take = INF;
        if (coins[i] <= amount) {
            take = 1 + dfs(i, amount - coins[i]); // i pe hi raho (unbounded)
        }
        
        memo[i][amount] = Math.min(skip, take);
        return memo[i][amount];
    }
}
```

**Key Insight:** Unbounded knapsack me item ko pick karne ke baad index increment nahi hota.

## 6. **Coin Change II (Leetcode 518)**

**Instinct:**  
Amount banane ke total ways chahiye. Coins unlimited. Ye **unbounded knapsack ka count variant** hai.

**Approach:**  
1. Memo[i][amount] = ways from index i to make amount
2. Base: amount == 0 → 1; i == n → 0
3. Ways = take (i pe hi raho) + skip (i+1)

```java
class Solution {
    int[] coins;
    int n;
    Integer[][] memo;
    
    public int change(int amount, int[] coins) {
        this.coins = coins;
        n = coins.length;
        memo = new Integer[n][amount + 1];
        return dfs(0, amount);
    }
    
    private int dfs(int i, int amount) {
        if (amount == 0) return 1;
        if (i == n) return 0;
        if (memo[i][amount] != null) return memo[i][amount];
        
        int skip = dfs(i + 1, amount);
        int take = 0;
        if (coins[i] <= amount) {
            take = dfs(i, amount - coins[i]); // unbounded - index same
        }
        
        memo[i][amount] = skip + take;
        return memo[i][amount];
    }
}
```

**Key Insight:** Count ways wala unbounded knapsack. Order of coins matter nahi karta.

## 7. **Combination Sum IV (Leetcode 377)**

**Instinct:**  
Target sum banane ke total ways, but **order matters** (permutations count hoti hain). Ye unbounded knapsack hai but with ordering.

**Approach:**  
1. Memo[target] = ways to make target
2. Har target ke liye, try karo saare coins
3. Base: target == 0 → 1

```java
class Solution {
    int[] nums;
    int[] memo;
    
    public int combinationSum4(int[] nums, int target) {
        this.nums = nums;
        memo = new int[target + 1];
        Arrays.fill(memo, -1);
        memo[0] = 1;
        return dfs(target);
    }
    
    private int dfs(int target) {
        if (target < 0) return 0;
        if (memo[target] != -1) return memo[target];
        
        int ways = 0;
        for (int num : nums) {
            ways += dfs(target - num);
        }
        memo[target] = ways;
        return ways;
    }
}
```

**Key Insight:** Order matters isliye 1D DP enough hai. Har amount ke liye saare coins try karo.

## 8. **Minimum Cost to Cut a Stick (Leetcode 1547)**

**Instinct:**  
Stick ko given cuts pe cut karna hai. Cost = length of stick. Order of cuts se total cost change hoti hai. Ye **optimal BST / matrix chain multiplication jaisa** hai - interval DP.

**Approach:**  
1. Cuts array me 0 aur n add karo, sort karo
2. Memo[i][j] = min cost to cut from cuts[i] to cuts[j]
3. Try karo har k as first cut

```java
class Solution {
    int[] cuts;
    Integer[][] memo;
    
    public int minCost(int n, int[] cuts) {
        this.cuts = new int[cuts.length + 2];
        this.cuts[0] = 0;
        this.cuts[cuts.length + 1] = n;
        for (int i = 0; i < cuts.length; i++) {
            this.cuts[i + 1] = cuts[i];
        }
        Arrays.sort(this.cuts);
        
        memo = new Integer[this.cuts.length][this.cuts.length];
        return dfs(0, this.cuts.length - 1);
    }
    
    private int dfs(int i, int j) {
        if (i + 1 >= j) return 0;
        if (memo[i][j] != null) return memo[i][j];
        
        int min = Integer.MAX_VALUE;
        for (int k = i + 1; k < j; k++) {
            int cost = (cuts[j] - cuts[i]) + dfs(i, k) + dfs(k, j);
            min = Math.min(min, cost);
        }
        
        memo[i][j] = min;
        return min;
    }
}
```

**Key Insight:** Ye knapsack nahi hai, but interval DP hai. Cuts ke order ka optimization.

## 9. **Profitable Schemes (Leetcode 879)**

**Instinct:**  
Crimes hain, har crime me profit aur members required. At least minProfit earn karna hai with at most n members. Ye **2D knapsack with constraint on minimum** hai.

**Approach:**  
1. Memo[i][members][profit] = ways from index i with members left and profit achieved
2. Profit ko minProfit se cap kardo (kyunki extra profit matter nahi karta)

```java
class Solution {
    int MOD = 1_000_000_007;
    int[] group, profit;
    int minProfit;
    int n;
    Integer[][][] memo;
    
    public int profitableSchemes(int n, int minProfit, int[] group, int[] profit) {
        this.group = group;
        this.profit = profit;
        this.minProfit = minProfit;
        this.n = n;
        memo = new Integer[group.length][n + 1][minProfit + 1];
        return dfs(0, n, 0);
    }
    
    private int dfs(int i, int members, int currProfit) {
        if (i == group.length) {
            return currProfit >= minProfit ? 1 : 0;
        }
        
        int profitCap = Math.min(currProfit, minProfit);
        if (memo[i][members][profitCap] != null) return memo[i][members][profitCap];
        
        // skip
        long ways = dfs(i + 1, members, currProfit);
        
        // take
        if (group[i] <= members) {
            ways += dfs(i + 1, members - group[i], currProfit + profit[i]);
        }
        
        memo[i][members][profitCap] = (int)(ways % MOD);
        return memo[i][members][profitCap];
    }
}
```

**Key Insight:** Profit ko minProfit pe cap karo taaki state space kam ho. 3D DP.

## 10. **Minimum Number of Refueling Stops (Leetcode 871) - Knapsack Variant**

**Instinct:**  
Stations pe fuel milta hai. Target tak pahunchne ke liye minimum stops. Ye **knapsack jaisa hai** - har station ko le sakte ho ya nahi, but order matter karta hai.

**Approach:**  
1. Memo[i][stops] = max fuel reachable at station i with given stops
2. Ya toh current station pe fuel lo ya skip

```java
class Solution {
    int[][] stations;
    int target;
    Long[][] memo;
    
    public int minRefuelStops(int target, int startFuel, int[][] stations) {
        this.stations = stations;
        this.target = target;
        memo = new Long[stations.length + 1][stations.length + 1];
        int ans = dfs(0, startFuel, 0);
        return ans >= Integer.MAX_VALUE / 2 ? -1 : ans;
    }
    
    private int dfs(int i, int fuel, int stops) {
        if (fuel >= target) return stops;
        if (i >= stations.length) return Integer.MAX_VALUE / 2;
        if (memo[i][stops] != null) return memo[i][stops].intValue();
        
        int res = Integer.MAX_VALUE / 2;
        
        // skip
        if (fuel >= stations[i][0]) {
            res = Math.min(res, dfs(i + 1, fuel, stops));
        }
        
        // take
        if (fuel >= stations[i][0]) {
            res = Math.min(res, dfs(i + 1, fuel + stations[i][1], stops + 1));
        }
        
        memo[i][stops] = (long)res;
        return res;
    }
}
```

**Key Insight:** 0/1 knapsack with condition - station tabhi le sakte ho jab fuel se waha pahunch sakte ho.

## 11. **Maximum Earnings From Taxi (Leetcode 2008)**

**Instinct:**  
Rides hain, har ride (start, end, tip). Ek ride leke doosri ride tabhi le sakte ho jab end <= next start. Ye **weighted interval scheduling** hai - knapsack jaisa hai.

**Approach:**  
1. Rides ko end time se sort karo
2. Memo[i] = max earning from index i
3. Ya toh current ride lo + next non-conflicting, ya skip

```java
class Solution {
    int[][] rides;
    int n;
    Long[] memo;
    
    public long maxTaxiEarnings(int n, int[][] rides) {
        this.rides = rides;
        Arrays.sort(rides, (a, b) -> a[1] - b[1]);
        memo = new Long[rides.length];
        return dfs(0);
    }
    
    private long dfs(int i) {
        if (i >= rides.length) return 0;
        if (memo[i] != null) return memo[i];
        
        int start = rides[i][0], end = rides[i][1], tip = rides[i][2];
        long earning = (end - start + tip);
        
        // find next non-conflicting ride
        int next = findNext(i);
        
        long take = earning + dfs(next);
        long skip = dfs(i + 1);
        
        memo[i] = Math.max(take, skip);
        return memo[i];
    }
    
    private int findNext(int i) {
        int lo = i + 1, hi = rides.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (rides[mid][0] >= rides[i][1]) hi = mid;
            else lo = mid + 1;
        }
        return lo;
    }
}
```

**Key Insight:** Weighted interval scheduling = knapsack with time constraints.

## 12. **Number of Ways to Form a Target String Given a Dictionary (Leetcode 1639)**

**Instinct:**  
Dictionary words se target string form karni hai. Har position pe koi bhi word se char le sakte ho. Ye **knapsack with counting** hai - har column se kitne characters available hain.

**Approach:**  
1. frequency[col][char] = kitne words me col pe char hai
2. Memo[i][j] = ways to form target from index i using column j

```java
class Solution {
    int MOD = 1_000_000_007;
    String target;
    int[][] freq;
    Integer[][] memo;
    
    public int numWays(String[] words, String target) {
        this.target = target;
        int m = words[0].length();
        int n = target.length();
        if (m < n) return 0;
        
        freq = new int[m][26];
        for (String word : words) {
            for (int j = 0; j < m; j++) {
                freq[j][word.charAt(j) - 'a']++;
            }
        }
        
        memo = new Integer[n][m];
        return dfs(0, 0);
    }
    
    private int dfs(int targetIdx, int col) {
        if (targetIdx == target.length()) return 1;
        if (col >= freq.length) return 0;
        if (memo[targetIdx][col] != null) return memo[targetIdx][col];
        
        int ch = target.charAt(targetIdx) - 'a';
        
        // skip current column
        long ways = dfs(targetIdx, col + 1);
        
        // use current column
        if (freq[col][ch] > 0) {
            ways += (long)freq[col][ch] * dfs(targetIdx + 1, col + 1);
        }
        
        memo[targetIdx][col] = (int)(ways % MOD);
        return memo[targetIdx][col];
    }
}
```

**Key Insight:** Knapsack jaisa - har column ko use karo ya skip, but count multiply hota hai.

## 13. **Knight Dialer (Leetcode 935) - Unbounded Knapsack Style**

**Instinct:**  
Phone keypad pe knight moves se n-digit numbers banane hain. Har position se kuch specific positions pe ja sakte ho. Ye **path counting with DP** hai - knapsack nahi but unbounded jaisa hai.

**Approach:**  
1. Memo[digit][remaining] = ways starting from digit with remaining steps
2. Base: remaining == 0 → 1
3. Sum over all next moves

```java
class Solution {
    int MOD = 1_000_000_007;
    int[][] moves = {
        {4,6}, {6,8}, {7,9}, {4,8}, {0,3,9}, {},
        {0,1,7}, {2,6}, {1,3}, {2,4}
    };
    Integer[][] memo;
    
    public int knightDialer(int n) {
        memo = new Integer[10][n];
        long total = 0;
        for (int digit = 0; digit <= 9; digit++) {
            total = (total + dfs(digit, n - 1)) % MOD;
        }
        return (int)total;
    }
    
    private int dfs(int digit, int remaining) {
        if (remaining == 0) return 1;
        if (memo[digit][remaining] != null) return memo[digit][remaining];
        
        long ways = 0;
        for (int next : moves[digit]) {
            ways = (ways + dfs(next, remaining - 1)) % MOD;
        }
        
        memo[digit][remaining] = (int)ways;
        return memo[digit][remaining];
    }
}
```

**Key Insight:** Ye graph traversal hai with memoization. Knapsack pattern jaisa hi hai - har state se next states explore karo.

## 14. **Number of Dice Rolls With Target Sum (Leetcode 1155)**

**Instinct:**  
n dice, har dice ke k faces. Target sum banana hai. Ye **unbounded knapsack count variant** hai - har dice ko ek baar use karte ho, but faces multiple options.

**Approach:**  
1. Memo[dice][target] = ways with remaining dice to reach target
2. Base: dice == 0 → target == 0 ? 1 : 0
3. Try all faces (1 to k)

```java
class Solution {
    int MOD = 1_000_000_007;
    int k;
    Integer[][] memo;
    
    public int numRollsToTarget(int n, int k, int target) {
        this.k = k;
        memo = new Integer[n + 1][target + 1];
        return dfs(n, target);
    }
    
    private int dfs(int dice, int target) {
        if (target < 0) return 0;
        if (dice == 0) return target == 0 ? 1 : 0;
        if (memo[dice][target] != null) return memo[dice][target];
        
        long ways = 0;
        for (int face = 1; face <= k; face++) {
            ways = (ways + dfs(dice - 1, target - face)) % MOD;
        }
        
        memo[dice][target] = (int)ways;
        return memo[dice][target];
    }
}
```

**Key Insight:** Bounded knapsack - har dice ko exactly ek baar use karte ho, but uske multiple options hain.

## 15. **Tallest Billboard (Leetcode 956) - Knapsack Variant**

**Instinct:**  
Do rods ka height difference maintain karte ho. Har rod ko left, right, ya discard. Ye **knapsack with difference tracking** hai.

**Approach:**  
1. Memo[i][diff] = max common height from index i with given difference
2. Try: add to left (diff increases), add to right (diff decreases), skip

```java
class Solution {
    int[] rods;
    Integer[][] memo;
    
    public int tallestBillboard(int[] rods) {
        this.rods = rods;
        memo = new Integer[rods.length][10001];
        return dfs(0, 0);
    }
    
    private int dfs(int i, int diff) {
        if (i == rods.length) return diff == 0 ? 0 : Integer.MIN_VALUE / 2;
        if (memo[i][diff + 5000] != null) return memo[i][diff + 5000];
        
        int skip = dfs(i + 1, diff);
        int left = rods[i] + dfs(i + 1, diff + rods[i]);
        int right = dfs(i + 1, Math.abs(diff - rods[i]));
        if (diff >= rods[i]) right += rods[i];
        
        int res = Math.max(skip, Math.max(left, right));
        memo[i][diff + 5000] = res;
        return res;
    }
}
```

**Key Insight:** Diff track karo (offset 5000 for negative). Har rod ko 3 options.

## 16. **Partition to K Equal Sum Subsets (Leetcode 698)**

**Instinct:**  
Array ko k subsets me divide karna hai jinka sum equal ho. Ye **multiple knapsack** hai - har subset ka sum = total/k.

**Approach:**  
1. Agar total % k != 0 return false
2. target = total / k
3. Memo[mask][subsetSum] with bitmask for used items

```java
class Solution {
    int[] nums;
    int target;
    Boolean[] memo;
    
    public boolean canPartitionKSubsets(int[] nums, int k) {
        this.nums = nums;
        int total = 0;
        for (int num : nums) total += num;
        if (total % k != 0) return false;
        
        target = total / k;
        Arrays.sort(nums);
        if (nums[nums.length - 1] > target) return false;
        
        memo = new Boolean[1 << nums.length];
        return dfs(0, 0, k);
    }
    
    private boolean dfs(int mask, int currSum, int remaining) {
        if (remaining == 0) return true;
        if (currSum == target) return dfs(mask, 0, remaining - 1);
        if (memo[mask] != null) return memo[mask];
        
        for (int i = 0; i < nums.length; i++) {
            if ((mask & (1 << i)) != 0) continue;
            if (currSum + nums[i] > target) continue;
            
            if (dfs(mask | (1 << i), currSum + nums[i], remaining)) {
                memo[mask] = true;
                return true;
            }
        }
        
        memo[mask] = false;
        return false;
    }
}
```

**Key Insight:** Bitmask DP for subset partition. Classic knapsack extension.

## 17. **Minimum Cost to Connect Sticks (Leetcode 1167)**

**Instinct:**  
Sticks ko jodne ki cost = sum of two sticks. Minimum total cost chahiye. Ye **Huffman coding / greedy** hai, but knapsack se relation nahi. Phir bhi pattern me daal raha hoon kyunki popular hai.

**Approach:**  
Min-heap use karo. Always join smallest two sticks.

```java
class Solution {
    public int connectSticks(int[] sticks) {
        PriorityQueue<Integer> pq = new PriorityQueue<>();
        for (int s : sticks) pq.offer(s);
        
        int cost = 0;
        while (pq.size() > 1) {
            int a = pq.poll();
            int b = pq.poll();
            int sum = a + b;
            cost += sum;
            pq.offer(sum);
        }
        return cost;
    }
}
```

**Key Insight:** Greedy, DP nahi. But interview me poochte hain.

## 18. **Minimum Cost to Merge Stones (Leetcode 1000)**

**Instinct:**  
Stones ko k groups me merge karna hai. Cost = sum of stones in group. Minimum total cost. Ye **knapsack + interval DP** hai.

**Approach:**  
1. Memo[i][j][k] = min cost to merge stones[i..j] into k piles
2. Base: if i==j → cost 0 if k==1 else INF
3. Try partition at mid

```java
class Solution {
    int[] prefix;
    int K;
    Integer[][][] memo;
    
    public int mergeStones(int[] stones, int k) {
        int n = stones.length;
        if ((n - 1) % (k - 1) != 0) return -1;
        
        this.K = k;
        prefix = new int[n + 1];
        for (int i = 0; i < n; i++) {
            prefix[i + 1] = prefix[i] + stones[i];
        }
        
        memo = new Integer[n][n][k + 1];
        return dfs(0, n - 1, 1);
    }
    
    private int dfs(int i, int j, int piles) {
        if (i == j) return piles == 1 ? 0 : Integer.MAX_VALUE / 2;
        if (memo[i][j][piles] != null) return memo[i][j][piles];
        
        if (piles == 1) {
            int val = dfs(i, j, K);
            if (val >= Integer.MAX_VALUE / 2) return Integer.MAX_VALUE / 2;
            return val + prefix[j + 1] - prefix[i];
        }
        
        int min = Integer.MAX_VALUE / 2;
        for (int mid = i; mid < j; mid++) {
            int left = dfs(i, mid, piles - 1);
            int right = dfs(mid + 1, j, 1);
            min = Math.min(min, left + right);
        }
        
        memo[i][j][piles] = min;
        return min;
    }
}
```

**Key Insight:** Interval DP with piles state. Knapsack pattern ka extension.

## 19. **Maximum Number of Achievable Transfer Requests (Leetcode 1601)**

**Instinct:**  
Building me transfer requests hain. Maximum requests satisfy karni hain such that net change of each building = 0. Ye **subset sum / knapsack** hai - choose subset of requests.

**Approach:**  
1. Bitmask over requests (2^n possibilities)
2. Check karo ki valid hai ya nahi

```java
class Solution {
    int n;
    int[][] requests;
    int max = 0;
    
    public int maximumRequests(int n, int[][] requests) {
        this.n = n;
        this.requests = requests;
        dfs(0, new int[n], 0);
        return max;
    }
    
    private void dfs(int idx, int[] balance, int count) {
        if (idx == requests.length) {
            for (int b : balance) {
                if (b != 0) return;
            }
            max = Math.max(max, count);
            return;
        }
        
        // skip request
        dfs(idx + 1, balance, count);
        
        // take request
        int from = requests[idx][0];
        int to = requests[idx][1];
        balance[from]--;
        balance[to]++;
        dfs(idx + 1, balance, count + 1);
        balance[from]++;
        balance[to]--;
    }
}
```

**Key Insight:** 0/1 knapsack on requests. Each request ya toh lo ya mat lo.

## 20. **Soup Servings (Leetcode 808)**

**Instinct:**  
Soup A aur B hai. Har operation me 4 possibilities. Probability find karni hai ki A finishes first. Ye **probability DP** hai - unbounded knapsack style.

**Approach:**  
1. Memo[a][b] = probability from (a,b) ml
2. Base: a <= 0 && b <= 0 → 0.5; a <= 0 → 1; b <= 0 → 0
3. Try all 4 operations

```java
class Solution {
    double[][] memo;
    
    public double soupServings(int n) {
        if (n >= 4800) return 1.0; // optimization
        int N = (int)Math.ceil(n / 25.0);
        memo = new double[N + 1][N + 1];
        return dfs(N, N);
    }
    
    private double dfs(int a, int b) {
        if (a <= 0 && b <= 0) return 0.5;
        if (a <= 0) return 1.0;
        if (b <= 0) return 0.0;
        if (memo[a][b] != 0) return memo[a][b];
        
        double prob = 0.25 * (
            dfs(a - 4, b) +
            dfs(a - 3, b - 1) +
            dfs(a - 2, b - 2) +
            dfs(a - 1, b - 3)
        );
        
        memo[a][b] = prob;
        return prob;
    }
}
```

**Key Insight:** Unbounded knapsack style with probabilities. N large hone pe optimization chahiye.
