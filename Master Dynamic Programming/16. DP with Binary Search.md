## 1. **Longest Increasing Subsequence (LIS) - O(n log n)**

**Instinct:**  
Classic LIS O(n²) hota hai, but binary search se O(n log n) me optimize kar sakte ho. Tails array maintain karte ho jisme tails[i] = smallest possible tail of increasing subsequence of length i+1.

**Approach:**  
1. tails array banayo
2. Har element ke liye, binary search se find karo first index j where tails[j] >= num
3. tails[j] = num
4. Answer = length of tails

```java
class Solution {
    public int lengthOfLIS(int[] nums) {
        int[] tails = new int[nums.length];
        int size = 0;
        
        for (int num : nums) {
            int left = 0, right = size;
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] < num) {
                    left = mid + 1;
                } else {
                    right = mid;
                }
            }
            tails[left] = num;
            if (left == size) size++;
        }
        
        return size;
    }
}
```

**Key Insight:** Patience sorting. Binary search finds position to place current number.

## 2. **Russian Doll Envelopes (Leetcode 354) - O(n log n)**

**Instinct:**  
Envelopes (w, h). Pehle width ascending sort karo, agar width same toh height descending. Phir height pe LIS O(n log n) apply karo.

**Approach:**  
1. Sort envelopes by width ascending, height descending
2. Extract heights array
3. Apply LIS O(n log n) on heights

```java
class Solution {
    public int maxEnvelopes(int[][] envelopes) {
        // Sort: width ascending, if same width then height descending
        Arrays.sort(envelopes, (a, b) -> {
            if (a[0] == b[0]) return b[1] - a[1];
            return a[0] - b[0];
        });
        
        // LIS on heights
        int[] tails = new int[envelopes.length];
        int size = 0;
        
        for (int[] env : envelopes) {
            int height = env[1];
            int left = 0, right = size;
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] < height) {
                    left = mid + 1;
                } else {
                    right = mid;
                }
            }
            tails[left] = height;
            if (left == size) size++;
        }
        
        return size;
    }
}
```

**Key Insight:** Sort one dimension, LIS on other. Same width wale kabhi fit nahi hote.

## 3. **Maximum Profit in Job Scheduling (Leetcode 1235) - O(n log n)**

**Instinct:**  
Jobs with start, end, profit. Max profit without overlapping. Sort by end time, then binary search to find next non-conflicting job.

**Approach:**  
1. Sort jobs by end time
2. Memo[i] = max profit from index i
3. For job i, find next job j where start >= end[i] using binary search
4. Profit = max(skip, take + profit[j])

```java
class Solution {
    int[][] jobs;
    int n;
    long[] memo;
    
    public int jobScheduling(int[] startTime, int[] endTime, int[] profit) {
        n = startTime.length;
        jobs = new int[n][3];
        for (int i = 0; i < n; i++) {
            jobs[i] = new int[]{startTime[i], endTime[i], profit[i]};
        }
        Arrays.sort(jobs, (a, b) -> a[1] - b[1]);
        
        memo = new long[n];
        Arrays.fill(memo, -1);
        
        return (int) dfs(0);
    }
    
    private long dfs(int i) {
        if (i >= n) return 0;
        if (memo[i] != -1) return memo[i];
        
        long skip = dfs(i + 1);
        
        int nextIndex = findNext(i);
        long take = jobs[i][2] + dfs(nextIndex);
        
        memo[i] = Math.max(skip, take);
        return memo[i];
    }
    
    private int findNext(int i) {
        int lo = i + 1, hi = n;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (jobs[mid][0] >= jobs[i][1]) {
                hi = mid;
            } else {
                lo = mid + 1;
            }
        }
        return lo;
    }
}
```

**Key Insight:** Binary search for next non-conflicting job. O(n log n) time.

## 4. **Minimum Difficulty of a Job Schedule (Leetcode 1335) - Binary Search on Answer**

**Instinct:**  
Jobs ko d days me partition karna hai. Har day ka difficulty = max job. Min sum of difficulties. Binary search on answer possible? Actually DP O(n²d) better.

**Approach:**  
DP with memoization, but binary search not directly applicable.

```java
class Solution {
    int[] jobDifficulty;
    int n;
    Integer[][] memo;
    
    public int minDifficulty(int[] jobDifficulty, int d) {
        this.jobDifficulty = jobDifficulty;
        n = jobDifficulty.length;
        if (n < d) return -1;
        
        memo = new Integer[n][d + 1];
        return dfs(0, d);
    }
    
    private int dfs(int i, int daysLeft) {
        if (i == n && daysLeft == 0) return 0;
        if (i == n || daysLeft == 0) return Integer.MAX_VALUE / 2;
        if (memo[i][daysLeft] != null) return memo[i][daysLeft];
        
        int maxSoFar = 0;
        int minDiff = Integer.MAX_VALUE / 2;
        
        for (int j = i; j <= n - daysLeft; j++) {
            maxSoFar = Math.max(maxSoFar, jobDifficulty[j]);
            int rest = dfs(j + 1, daysLeft - 1);
            minDiff = Math.min(minDiff, maxSoFar + rest);
        }
        
        memo[i][daysLeft] = minDiff;
        return minDiff;
    }
}
```

**Key Insight:** Interval DP, not binary search. Included for pattern recognition.

## 5. **Split Array Largest Sum (Leetcode 410)**

**Instinct:**  
Array ko k subarrays me divide karna hai. Minimize the largest subarray sum. Binary search on answer.

**Approach:**  
1. Binary search on max sum
2. Check if possible to split array into k subarrays with each sum <= mid
3. Greedy check: keep adding to current subarray until sum > mid

```java
class Solution {
    public int splitArray(int[] nums, int k) {
        int left = 0, right = 0;
        for (int num : nums) {
            left = Math.max(left, num);
            right += num;
        }
        
        int ans = right;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (canSplit(nums, k, mid)) {
                ans = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return ans;
    }
    
    private boolean canSplit(int[] nums, int k, int maxSum) {
        int count = 1;
        int sum = 0;
        for (int num : nums) {
            if (sum + num > maxSum) {
                count++;
                sum = num;
                if (count > k) return false;
            } else {
                sum += num;
            }
        }
        return true;
    }
}
```

**Key Insight:** Binary search on maximum subarray sum. Greedy validation O(n).

## 6. **Koko Eating Bananas (Leetcode 875)**

**Instinct:**  
Koko bananas khata hai. Minimum speed k such that saari bananas h hours me khatam ho. Binary search on speed.

**Approach:**  
1. Binary search on speed from 1 to max pile
2. Check if speed works: total time = sum(ceil(pile/speed))

```java
class Solution {
    public int minEatingSpeed(int[] piles, int h) {
        int left = 1, right = 0;
        for (int pile : piles) right = Math.max(right, pile);
        
        int ans = right;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (canEat(piles, h, mid)) {
                ans = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return ans;
    }
    
    private boolean canEat(int[] piles, int h, int speed) {
        int hours = 0;
        for (int pile : piles) {
            hours += (pile + speed - 1) / speed;
            if (hours > h) return false;
        }
        return true;
    }
}
```

**Key Insight:** Binary search on speed. Ceil division formula.

## 7. **Capacity to Ship Packages Within D Days (Leetcode 1011)**

**Instinct:**  
Packages ko d days me ship karna hai. Minimum capacity. Binary search on capacity.

**Approach:**  
1. Binary search between max weight and total weight
2. Check if capacity works: count days needed

```java
class Solution {
    public int shipWithinDays(int[] weights, int days) {
        int left = 0, right = 0;
        for (int w : weights) {
            left = Math.max(left, w);
            right += w;
        }
        
        int ans = right;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (canShip(weights, days, mid)) {
                ans = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return ans;
    }
    
    private boolean canShip(int[] weights, int days, int capacity) {
        int currentLoad = 0;
        int daysNeeded = 1;
        for (int w : weights) {
            if (currentLoad + w > capacity) {
                daysNeeded++;
                currentLoad = w;
                if (daysNeeded > days) return false;
            } else {
                currentLoad += w;
            }
        }
        return true;
    }
}
```

**Key Insight:** Binary search on capacity. Greedy check.

## 8. **Minimum Number of Days to Make m Bouquets (Leetcode 1482)**

**Instinct:**  
Bloom day array. m bouquets, each k adjacent flowers. Min days. Binary search on day.

**Approach:**  
1. Binary search between min and max bloom day
2. Check if day works: count adjacent flowers bloomed

```java
class Solution {
    public int minDays(int[] bloomDay, int m, int k) {
        if (m * k > bloomDay.length) return -1;
        
        int left = Integer.MAX_VALUE, right = Integer.MIN_VALUE;
        for (int day : bloomDay) {
            left = Math.min(left, day);
            right = Math.max(right, day);
        }
        
        int ans = right;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (canMake(bloomDay, m, k, mid)) {
                ans = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return ans;
    }
    
    private boolean canMake(int[] bloomDay, int m, int k, int day) {
        int bouquets = 0;
        int flowers = 0;
        for (int bd : bloomDay) {
            if (bd <= day) {
                flowers++;
                if (flowers == k) {
                    bouquets++;
                    flowers = 0;
                }
            } else {
                flowers = 0;
            }
        }
        return bouquets >= m;
    }
}
```

**Key Insight:** Binary search on day. Count consecutive flowers.

Array nums, divisor d. Sum of ceil(nums[i]/d) <= threshold. Min divisor. Binary search on divisor.

## 10. **Magnetic Force Between Two Balls (Leetcode 1552)**

**Instinct:**  
Position array. Place m balls. Maximize minimum distance between balls. Binary search on distance.

**Approach:**  
1. Sort positions
2. Binary search on minimum distance
3. Greedy: place balls at least 'mid' apart

```java
class Solution {
    public int maxDistance(int[] position, int m) {
        Arrays.sort(position);
        int left = 1;
        int right = position[position.length - 1] - position[0];
        
        int ans = 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (canPlace(position, m, mid)) {
                ans = mid;
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        return ans;
    }
    
    private boolean canPlace(int[] position, int m, int dist) {
        int count = 1;
        int lastPos = position[0];
        for (int i = 1; i < position.length; i++) {
            if (position[i] - lastPos >= dist) {
                count++;
                lastPos = position[i];
                if (count >= m) return true;
            }
        }
        return count >= m;
    }
}
```

**Key Insight:** Binary search on minimum distance. Greedy placement.

## 11. **Minimum Time to Complete Trips (Leetcode 2187)**

**Instinct:**  
Buses with trip times. Total trips >= target. Minimum time. Binary search on time.

**Approach:**  
1. Binary search on time
2. Check: sum(time / busTime) >= target

```java
class Solution {
    public long minimumTime(int[] time, int totalTrips) {
        long left = 1;
        long right = (long) time[0] * totalTrips;
        for (int t : time) right = Math.min(right, (long) t * totalTrips);
        right = (long) time[0] * totalTrips;
        
        long ans = right;
        while (left <= right) {
            long mid = left + (right - left) / 2;
            if (totalTripsCount(time, mid) >= totalTrips) {
                ans = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return ans;
    }
    
    private long totalTripsCount(int[] time, long givenTime) {
        long trips = 0;
        for (int t : time) {
            trips += givenTime / t;
            if (trips >= Integer.MAX_VALUE) break;
        }
        return trips;
    }
}
```

**Key Insight:** Binary search on time. Sum of floor division.

## 12. **Maximum Running Time of N Computers (Leetcode 2141)**

**Instinct:**  
Batteries, n computers. Max minutes all computers can run. Binary search on minutes.

**Approach:**  
1. Binary search on minutes
2. Check: sum(min(battery, minutes)) >= n * minutes

```java
class Solution {

        long left = 1;
        long right = 0;
        for (int b : batteries) right += b;
        right = right / n;

                ans = mid;
                left = mid + 1;

                right = mid - 1;

        long total = 0;
        for (int b : batteries) {

        }

    }
}
```

## 14. **Maximum Number of Events That Can Be Attended (Leetcode 1353)**

**Instinct:**  
Events with start and end. Max events attend. Greedy + priority queue, not binary search.

**Approach:**  
1. Sort by start time
2. Use min-heap for end times

```java
class Solution {
    public int maxEvents(int[][] events) {
        Arrays.sort(events, (a, b) -> a[0] - b[0]);
        PriorityQueue<Integer> pq = new PriorityQueue<>();
        int i = 0, n = events.length;
        int day = 0;
        int count = 0;
        
        while (i < n || !pq.isEmpty()) {
            if (pq.isEmpty()) day = events[i][0];
            while (i < n && events[i][0] <= day) {
                pq.offer(events[i][1]);
                i++;
            }
            pq.poll();
            count++;
            day++;
            while (!pq.isEmpty() && pq.peek() < day) {
                pq.poll();
            }
        }
        return count;
    }
}
```

**Key Insight:** Greedy attend earliest ending event. Not binary search.

## 15. **Maximum Number of Visible Points (Leetcode 1610)**

**Instinct:**  
Points with angles. Max points visible from location. Binary search on angle window.

**Approach:**  
1. Convert points to angles
2. Binary search on number of points in window

```java
class Solution {
    public int visiblePoints(List<List<Integer>> points, int angle, List<Integer> location) {
        List<Double> angles = new ArrayList<>();
        int same = 0;
        int x = location.get(0), y = location.get(1);
        
        for (List<Integer> p : points) {
            int px = p.get(0), py = p.get(1);
            if (px == x && py == y) {
                same++;
            } else {
                double ang = Math.atan2(py - y, px - x) * 180 / Math.PI;
                angles.add(ang);
            }
        }
        
        Collections.sort(angles);
        int n = angles.size();
        List<Double> doubleAngles = new ArrayList<>();
        for (double a : angles) {
            doubleAngles.add(a);
            doubleAngles.add(a + 360);
        }
        
        int max = 0;
        int j = 0;
        for (int i = 0; i < n; i++) {
            while (j < doubleAngles.size() && doubleAngles.get(j) - doubleAngles.get(i) <= angle) {
                j++;
            }
            max = Math.max(max, j - i);
        }
        
        return max + same;
    }
}
```

**Key Insight:** Two-pointer on angles. Not pure binary search.

## 16. **Longest Increasing Subsequence with Binary Search (Detailed)**

**Instinct:**  
Patience sorting algorithm. Tails array stores smallest possible tail for each length.

**Approach:**  
1. tails[i] = smallest tail of increasing subsequence of length i+1
2. Binary search to find position for current number

```java
class Solution {
    public int lengthOfLIS(int[] nums) {
        int[] tails = new int[nums.length];
        int size = 0;
        
        for (int num : nums) {
            int pos = binarySearch(tails, size, num);
            tails[pos] = num;
            if (pos == size) size++;
        }
        
        return size;
    }
    
    private int binarySearch(int[] tails, int size, int target) {
        int left = 0, right = size;
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (tails[mid] < target) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }
        return left;
    }
}
```

**Key Insight:** Binary search for first index where tails[mid] >= target.

## 17. **Minimum Number of Operations to Make Array Continuous (Leetcode 2009)**

**Instinct:**  
Array me changes to make it continuous (max - min + 1 = n). Binary search on window.

**Approach:**  
1. Sort unique values
2. For each left, find right using binary search

```java
class Solution {
    public int minOperations(int[] nums) {
        int n = nums.length;
        int[] unique = Arrays.stream(nums).distinct().sorted().toArray();
        int m = unique.length;
        
        int maxLen = 1;
        int j = 0;
        for (int i = 0; i < m; i++) {
            while (j < m && unique[j] < unique[i] + n) {
                j++;
            }
            maxLen = Math.max(maxLen, j - i);
        }
        
        return n - maxLen;
    }
}
```

**Key Insight:** Two-pointer with sorted unique values.

## 18. **Find K-th Smallest Pair Distance (Leetcode 719)**

**Instinct:**  
Pairs ka distance. Find k-th smallest. Binary search on distance.

**Approach:**  
1. Sort array
2. Binary search on distance
3. Count pairs with distance <= mid using two pointers

```java
class Solution {
    public int smallestDistancePair(int[] nums, int k) {
        Arrays.sort(nums);
        int left = 0;
        int right = nums[nums.length - 1] - nums[0];
        
        int ans = right;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (countPairs(nums, mid) >= k) {
                ans = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return ans;

    private int countPairs(int[] nums, int maxDist) {
        int count = 0;
        int left = 0;
        for (int right = 0; right < nums.length; right++) {
            while (nums[right] - nums[left] > maxDist) {
                left++;
            }
            count += right - left;
        }
        return count;
    }
}
```

**Key Insight:** Binary search on distance. Two-pointer counting.

## 19. **Maximum Sum of Two Non-Overlapping Subarrays (Leetcode 1031)**

**Instinct:**  
Two non-overlapping subarrays of given lengths. Max sum. Prefix sum + binary search.

**Approach:**  
1. Prefix sum array
2. Try first subarray at i, second after i + len

```java
class Solution {
    public int maxSumTwoNoOverlap(int[] nums, int firstLen, int secondLen) {
        return Math.max(helper(nums, firstLen, secondLen), helper(nums, secondLen, firstLen));
    }
    
    private int helper(int[] nums, int len1, int len2) {
        int n = nums.length;
        int[] prefix = new int[n + 1];
        for (int i = 0; i < n; i++) {
            prefix[i + 1] = prefix[i] + nums[i];
        }
        
        int max = 0;
        int maxFirst = 0;
        for (int i = len1; i <= n - len2; i++) {
            maxFirst = Math.max(maxFirst, prefix[i] - prefix[i - len1]);
            max = Math.max(max, maxFirst + prefix[i + len2] - prefix[i]);
        }
        return max;
    }
}
```

**Key Insight:** Not binary search, but sliding window. Included for completeness.

## 20. **Maximum Distance in Arrays (Leetcode 624)**

**Instinct:**  
Multiple arrays. Pick one number from each, maximize difference. Not binary search.

**Approach:**  
1. Track global min and max
2. For each array, compute potential max distance

```java
class Solution {
    public int maxDistance(List<List<Integer>> arrays) {
        int minVal = arrays.get(0).get(0);
        int maxVal = arrays.get(0).get(arrays.get(0).size() - 1);
        int maxDist = 0;
        
        for (int i = 1; i < arrays.size(); i++) {
            List<Integer> arr = arrays.get(i);

            int currMax = arr.get(arr.size() - 1);
            
            maxDist = Math.max(maxDist, Math.abs(currMax - minVal));
            maxDist = Math.max(maxDist, Math.abs(maxVal - currMin));
            
            minVal = Math.min(minVal, currMin);
            maxVal = Math.max(maxVal, currMax);
        }
        
        return maxDist;
    }
}
```

## 🎯 Key Takeaways for DP with Binary Search Pattern

### 1. **Two Main Use Cases:**

**Case 1: Optimization using Binary Search on Answer**
```java
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (check(mid)) {
        ans = mid;
        right = mid - 1; // or left = mid + 1
    } else {
        left = mid + 1; // or right = mid - 1
    }
}
```

**Case 2: Binary Search within DP Transition**
```java
// Find next index with binary search
int nextIndex = binarySearch(jobs, endTime);
dp[i] = max(skip, take + dp[nextIndex]);
```

### 2. **Common Check Functions:**

| Problem | Check Function |
|---------|----------------|
| Split Array | Greedy partition count <= k |
| Koko Eating | Sum(ceil(pile/speed)) <= h |
| Ship Capacity | Days needed <= given days |
| Magnetic Force | Can place m balls with distance >= mid |

### 3. **Binary Search Patterns:**

```
                    Binary Search in DP
                           │
        ┌──────────────────┼──────────────────┐
        │                                     │
    On Answer                          On Index
        │                                     │
    ┌───┼───┐                           ┌───┼───┐
    │   │   │                           │   │   │

```

### 4. **Complexity Comparison:**

| Problem | O(n²) DP | O(n log n) Binary Search |
|---------|----------|--------------------------|
| LIS | O(n²) | O(n log n) |
| Russian Doll | O(n²) | O(n log n) |
| Job Scheduling | O(n²) | O(n log n) |
| Split Array | O(n²k) | O(n log sum) |

### 5. **When to use Binary Search on Answer:**
- Need to find minimum/maximum value satisfying condition
- Condition is monotonic (if x works, all larger/smaller work)
- Can check feasibility in O(n) or better

### 6. **Binary Search Template for Minimization:**
```java
int left = minPossible, right = maxPossible;
int ans = right;
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (isPossible(mid)) {
        ans = mid;
        right = mid - 1;  // try smaller
    } else {
        left = mid + 1;   // try larger
    }
}
return ans;
```

### 7. **Binary Search Template for Maximization:**
```java
int left = minPossible, right = maxPossible;
int ans = left;
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (isPossible(mid)) {
        ans = mid;
        left = mid + 1;   // try larger
    } else {
        right = mid - 1;  // try smaller
    }
}
return ans;
```

## 📝 Key Formulas to Remember

| Operation | Formula |
|-----------|---------|
| Ceil Division | `(a + b - 1) / b` |
| LIS Binary Search | `tails[mid] < num ? left = mid+1 : right = mid` |
| Job Scheduling Next | Binary search for `start >= end[i]` |
| Split Array Check | Greedy: add until sum > mid |
| Distance Count | `count += right - left` for sorted array |
