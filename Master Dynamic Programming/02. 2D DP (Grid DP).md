## 1. **Minimum Path Sum with Obstacles (Unique Paths II with min sum)**

**Instinct:**  
Grid me (0,0) se (m-1,n-1) tak jaana hai, sirf right ya down. Kuch cells blocked hain (value -1). Minimum sum path find karna hai. Socho - tum current cell pe ho, answer = current cell value + min( right ka answer, down ka answer ). Agar blocked cell hai toh infinite return karo.

**Approach:**  
1. Memo[m][n] array banayenge, -1 se initialize
2. dfs(i,j) function:
   - Agar out of bounds ya blocked → return INF
   - Agar (m-1,n-1) pe ho → return grid[i][j]
   - Agar memoized hai toh return
   - Right = dfs(i, j+1), Down = dfs(i+1, j)
   - Answer = grid[i][j] + Math.min(right, down)
   - Memoize karo

```java
class Solution {
    int INF = Integer.MAX_VALUE / 2;
    
    public int minPathSumWithObstacles(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        Integer[][] memo = new Integer[m][n];
        int ans = dfs(grid, 0, 0, m, n, memo);
        return ans >= INF ? -1 : ans;
    }
    
    private int dfs(int[][] grid, int i, int j, int m, int n, Integer[][] memo) {
        // Out of bounds ya blocked
        if (i >= m || j >= n || grid[i][j] == -1) return INF;
        
        // Destination pe pahunch gaye
        if (i == m - 1 && j == n - 1) return grid[i][j];
        
        if (memo[i][j] != null) return memo[i][j];
        
        int right = dfs(grid, i, j + 1, m, n, memo);
        int down = dfs(grid, i + 1, j, m, n, memo);
        
        int ans = grid[i][j] + Math.min(right, down);
        memo[i][j] = ans;
        return ans;
    }
}
```

**Key Insight:** INF use karo taaki overflow na ho. Blocked cells se recursion ko rok do.

## 2. **Cherry Pickup (Leetcode 741)**

**Instinct:**  
Ek bande ko (0,0) se (n-1,n-1) aur wapas (0,0) jaana hai, maximum cherries collect karni hai. But tricky part: wapas aate time bhi cherries le sakte ho. Isko solve karne ka smart tareeka: **do log ek saath (0,0) se (n-1,n-1) ja rahe hain** (r1,c1) aur (r2,c2). Since dono ek saath ek step chalenge, r1+c1 == r2+c2 hamesha.

**Approach:**  
1. Memo[r1][c1][r2] use karo (c2 = r1+c1-r2 se nikal sakte ho)
2. Base case: agar koi out of bounds ya -1 cell pe hai toh -INF return
3. Agar destination pe ho toh grid value return (agar dono same cell hai toh ek baar hi add karo)
4. 4 possibilities: (down,down), (down,right), (right,down), (right,right)
5. Max of 4 + current cherries

```java
class Solution {
    int n;
    int[][][] memo;
    int[][] grid;
    
    public int cherryPickup(int[][] grid) {
        this.grid = grid;
        n = grid.length;
        memo = new int[n][n][n];
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
                Arrays.fill(memo[i][j], Integer.MIN_VALUE);
        
        return Math.max(0, dfs(0, 0, 0));
    }
    
    private int dfs(int r1, int c1, int r2) {
        int c2 = r1 + c1 - r2;
        
        // Out of bounds ya -1 cell
        if (r1 >= n || c1 >= n || r2 >= n || c2 >= n) return Integer.MIN_VALUE;
        if (grid[r1][c1] == -1 || grid[r2][c2] == -1) return Integer.MIN_VALUE;
        
        // Destination
        if (r1 == n-1 && c1 == n-1) return grid[r1][c1];
        
        if (memo[r1][c1][r2] != Integer.MIN_VALUE) return memo[r1][c1][r2];
        
        int cherries = grid[r1][c1];
        if (r1 != r2 || c1 != c2) cherries += grid[r2][c2];
        
        // 4 possibilities
        int maxNext = Math.max(
            Math.max(dfs(r1+1, c1, r2+1), dfs(r1+1, c1, r2)),
            Math.max(dfs(r1, c1+1, r2+1), dfs(r1, c1+1, r2))
        );
        
        cherries += maxNext;
        memo[r1][c1][r2] = cherries;
        return cherries;
    }
}
```

**Key Insight:** Do logon ki movement ek saath track karo. r1+c1 == r2+c2 invariant maintain karo.

## 3. **Maximum Number of Points with Cost (Leetcode 1937)**

**Instinct:**  
Har row se ek column choose karna hai. Agar adjacent rows me columns different hain toh cost subtract karo |col1 - col2|. Maximum total points nikaalna hai. Isme brute force O(m*n^2) hoga, optimize karna hai O(m*n) me.

**Approach:**  
1. Memo[row][col] = max points from (row,col) to bottom
2. Current row ke liye, left se right sweep karo: leftMax[col] = max(leftMax[col-1] - 1, dp[col])
3. Right se left sweep karo: rightMax[col] = max(rightMax[col+1] - 1, dp[col])
4. Next row dp = max(leftMax[col], rightMax[col]) + grid[row][col]

```java
class Solution {
    long[][] memo;
    int m, n;
    
    public long maxPoints(int[][] points) {
        m = points.length;
        n = points[0].length;
        memo = new long[m][n];
        for (int i = 0; i < m; i++) Arrays.fill(memo[i], -1);
        
        long ans = 0;
        for (int col = 0; col < n; col++) {
            ans = Math.max(ans, dfs(points, 0, col));
        }
        return ans;
    }
    
    private long dfs(int[][] points, int row, int col) {
        if (row == m - 1) return points[row][col];
        if (memo[row][col] != -1) return memo[row][col];
        
        long maxNext = 0;
        for (int nextCol = 0; nextCol < n; nextCol++) {
            long next = dfs(points, row + 1, nextCol);
            long penalty = Math.abs(col - nextCol);
            maxNext = Math.max(maxNext, next - penalty);
        }
        
        memo[row][col] = points[row][col] + maxNext;
        return memo[row][col];
    }
}
```

**Key Insight:** Memoization se O(m*n^2) hoga, lekin interview me ye acceptable hai. Optimized O(m*n) ke liye left-right sweep use hota hai.

## 4. **Dungeon Game (Leetcode 174)**

**Instinct:**  
Prince (0,0) se princess (m-1,n-1) tak jaana hai. Har cell me +ve (health gain) ya -ve (damage) ho sakta hai. Minimum starting health chahiye ki kabhi health 0 na ho. **Yaha reverse DP karna easy hai** - destination se start tak.

**Approach:**  
1. Memo[i][j] = min health needed from (i,j) to destination
2. Base case: memo[m-1][n-1] = max(1, 1 - grid[m-1][n-1])
3. For other cells: need = min(right, down) - grid[i][j]
4. Agar need <= 0 toh 1 return (kyunki 1 health se bhi survive kar sakte ho)
5. Warna need return

```java
class Solution {
    Integer[][] memo;
    int m, n;
    
    public int calculateMinimumHP(int[][] dungeon) {
        m = dungeon.length;
        n = dungeon[0].length;
        memo = new Integer[m][n];
        return dfs(dungeon, 0, 0);
    }
    
    private int dfs(int[][] grid, int i, int j) {
        if (i >= m || j >= n) return Integer.MAX_VALUE;
        if (i == m-1 && j == n-1) {
            return Math.max(1, 1 - grid[i][j]);
        }
        if (memo[i][j] != null) return memo[i][j];
        
        int right = dfs(grid, i, j+1);
        int down = dfs(grid, i+1, j);
        int minNext = Math.min(right, down);
        int need = minNext - grid[i][j];
        
        int res = need <= 0 ? 1 : need;
        memo[i][j] = res;
        return res;
    }
}
```

**Key Insight:** Reverse DP karo. Agar need <= 0 matlab current cell itna bada +ve hai ki 1 health se bhi kaam chalega.

## 5. **Maximum Sum of 3 Non-Overlapping Subarrays (Leetcode 689)**

**Instinct:**  
Array me 3 non-overlapping subarrays of length k dhundhni hai jinka sum maximum ho. 2D DP: dp[i][count] = max sum using first i elements with 'count' subarrays.

**Approach:**  
1. Prefix sum array banao for O(1) subarray sum
2. Memo[index][count] = max sum from index to end with 'count' subarrays remaining
3. Ya toh current index skip karo, ya yaha se subarray le lo (length k)
4. Count == 0 → 0 return

```java
class Solution {
    int[] prefix;
    int k;
    int[][] memo;
    int[] result;
    
    public int[] maxSumOfThreeSubarrays(int[] nums, int k) {
        this.k = k;
        int n = nums.length;
        prefix = new int[n+1];
        for (int i = 0; i < n; i++) {
            prefix[i+1] = prefix[i] + nums[i];
        }
        
        memo = new int[n][4];
        for (int i = 0; i < n; i++) Arrays.fill(memo[i], -1);
        
        result = new int[3];
        dfs(nums, 0, 3);
        return result;
    }
    
    private int dfs(int[] nums, int start, int count) {
        if (count == 0) return 0;
        if (start > nums.length - k) return 0;
        if (memo[start][count] != -1) return memo[start][count];
        
        int takeSum = prefix[start + k] - prefix[start];
        int take = takeSum + dfs(nums, start + k, count - 1);
        
        int skip = dfs(nums, start + 1, count);
        
        if (take >= skip) {
            memo[start][count] = take;
            if (count == 3) result[3-count] = start;
        } else {
            memo[start][count] = skip;
        }
        return memo[start][count];
    }
}
```

**Key Insight:** Memoization ke saath indices store karna important hai for reconstructing path.

## 6. **Minimum Falling Path Sum II (Leetcode 1289)**

**Instinct:**  
Har row se ek column choose karna hai, adjacent rows me same column nahi le sakte. Minimum sum path. Isme brute force O(m*n^2) hoga, lekin har row ke liye top 2 minimums store karke O(m*n) kar sakte ho.

**Approach:**  
1. Memo[row][col] = min sum from (row,col) to bottom
2. Base case: last row → grid[row][col]
3. Current row ke liye, next row ka minimum dhundho except same column
4. Memoization use karo

```java
class Solution {
    int[][] memo;
    int m, n;
    
    public int minFallingPathSum(int[][] grid) {
        m = grid.length;
        n = grid[0].length;
        memo = new int[m][n];
        for (int i = 0; i < m; i++) Arrays.fill(memo[i], Integer.MAX_VALUE);
        
        int ans = Integer.MAX_VALUE;
        for (int col = 0; col < n; col++) {
            ans = Math.min(ans, dfs(grid, 0, col));
        }
        return ans;
    }
    
    private int dfs(int[][] grid, int row, int col) {
        if (row == m - 1) return grid[row][col];
        if (memo[row][col] != Integer.MAX_VALUE) return memo[row][col];
        
        int minNext = Integer.MAX_VALUE;
        for (int nextCol = 0; nextCol < n; nextCol++) {
            if (nextCol == col) continue;
            minNext = Math.min(minNext, dfs(grid, row + 1, nextCol));
        }
        
        memo[row][col] = grid[row][col] + minNext;
        return memo[row][col];
    }
}
```

**Key Insight:** Har column ke liye next row ka min find karte waqt O(n^2) lagta hai. Optimized me top-2 minimums store karte hain.

## 7. **Largest Rectangle in Histogram (as Grid DP)**

**Instinct:**  
Binary matrix me sabse bada rectangle containing only 1's. Har row ko histogram ki tarah treat karo: heights[j] = agar cell 1 hai toh previous height+1 else 0. Har row ke liye largest rectangle in histogram find karo.

**Approach:**  
1. heights array maintain karo
2. Har row ke liye heights update karo (agar 1 hai toh +1 else 0)
3. Har row ke liye largestRectangleArea(heights) call karo using stack
4. Max track karo

```java
class Solution {
    int max = 0;
    
    public int maximalRectangle(char[][] matrix) {
        if (matrix.length == 0) return 0;
        int m = matrix.length, n = matrix[0].length;
        int[] heights = new int[n];
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (matrix[i][j] == '1') heights[j]++;
                else heights[j] = 0;
            }
            max = Math.max(max, largestRectangleArea(heights));
        }
        return max;
    }
    
    private int largestRectangleArea(int[] heights) {
        Stack<Integer> stack = new Stack<>();
        int maxArea = 0;
        for (int i = 0; i <= heights.length; i++) {
            int h = (i == heights.length) ? 0 : heights[i];
            while (!stack.isEmpty() && h < heights[stack.peek()]) {
                int height = heights[stack.pop()];
                int width = stack.isEmpty() ? i : i - stack.peek() - 1;
                maxArea = Math.max(maxArea, height * width);
            }
            stack.push(i);
        }
        return maxArea;
    }
}
```

**Key Insight:** Histogram method O(n) per row, total O(m*n). Stack se previous smaller element concept use hota hai.

## 8. **Count Square Submatrices with All Ones (Leetcode 1277)**

**Instinct:**  
Matrix me kitne squares hain jinka sab cells 1 hain. DP[i][j] = side length of largest square ending at (i,j). Agar cell 1 hai toh dp[i][j] = 1 + min(up, left, diagonal). Sum of all dp[i][j] = total squares.

**Approach:**  
1. Memo[i][j] = side length of largest square ending at (i,j)
2. Base: first row/col pe agar 1 hai toh 1 else 0
3. Recurrence: if grid[i][j] == 1 → 1 + min(top, left, diag)
4. Total sum += dp[i][j]

```java
class Solution {
    int[][] memo;
    int m, n;
    int total = 0;
    
    public int countSquares(int[][] matrix) {
        m = matrix.length;
        n = matrix[0].length;
        memo = new int[m][n];
        for (int i = 0; i < m; i++) Arrays.fill(memo[i], -1);
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                total += dfs(matrix, i, j);
            }
        }
        return total;
    }
    
    private int dfs(int[][] matrix, int i, int j) {
        if (i >= m || j >= n) return 0;
        if (matrix[i][j] == 0) return 0;
        if (memo[i][j] != -1) return memo[i][j];
        
        int right = dfs(matrix, i, j+1);
        int down = dfs(matrix, i+1, j);
        int diag = dfs(matrix, i+1, j+1);
        
        int side = 1 + Math.min(right, Math.min(down, diag));
        memo[i][j] = side;
        return side;
    }
}
```

**Key Insight:** Square ki property - agar (i,j) pe square bana hai toh uske right, down, diagonal bhi squares hone chahiye.

## 9. **Maximum Profit from Trading Stocks (with transaction fee / cooldown)**

**Instinct:**  
Har din ya toh buy, sell, ya kuch mat karo. 2D DP with states: [day][holding] (0 = no stock, 1 = holding stock). Memoization se solve karo.

**Approach (with cooldown):**  
1. Memo[day][holding] = max profit from day to end
2. If holding == 1: either sell today ya hold
3. If holding == 0: either buy today (cooldown check) ya skip
4. Cooldown: sell ke baad ek din wait

```java
class Solution {
    int[] prices;
    Integer[][] memo;
    
    public int maxProfitWithCooldown(int[] prices) {
        this.prices = prices;
        memo = new Integer[prices.length][2];
        return dfs(0, 0);
    }
    
    private int dfs(int day, int holding) {
        if (day >= prices.length) return 0;
        if (memo[day][holding] != null) return memo[day][holding];
        
        if (holding == 1) {
            // sell today or hold
            int sell = prices[day] + dfs(day + 2, 0); // cooldown
            int hold = dfs(day + 1, 1);
            memo[day][holding] = Math.max(sell, hold);
        } else {
            // buy today or skip
            int buy = -prices[day] + dfs(day + 1, 1);
            int skip = dfs(day + 1, 0);
            memo[day][holding] = Math.max(buy, skip);
        }
        return memo[day][holding];
    }
}
```

**Key Insight:** States (day, holding) are enough. Cooldown ke liye sell ke baad day+2.

## 10. **Longest Increasing Path in a Matrix (Leetcode 329)**

**Instinct:**  
Matrix me longest path jisme har next cell > current cell. 4 directions me move kar sakte ho. Memoization DFS: dp[i][j] = longest path starting from (i,j). Agar adjacent cell > current, toh 1 + dp[adjacent].

**Approach:**  
1. Memo[i][j] = longest path from (i,j)
2. DFS: try all 4 directions where next > current
3. Path length = 1 + max(neighbors)
4. Memoize karo

```java
class Solution {
    int[][] matrix;
    int m, n;
    int[][] memo;
    int[][] dirs = {{0,1},{1,0},{0,-1},{-1,0}};
    
    public int longestIncreasingPath(int[][] matrix) {
        this.matrix = matrix;
        m = matrix.length;
        n = matrix[0].length;
        memo = new int[m][n];
        
        int maxLen = 0;
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                maxLen = Math.max(maxLen, dfs(i, j));
            }
        }
        return maxLen;
    }
    
    private int dfs(int i, int j) {
        if (memo[i][j] != 0) return memo[i][j];
        
        int max = 1;
        for (int[] dir : dirs) {
            int ni = i + dir[0];
            int nj = j + dir[1];
            if (ni >= 0 && ni < m && nj >= 0 && nj < n && matrix[ni][nj] > matrix[i][j]) {
                max = Math.max(max, 1 + dfs(ni, nj));
            }
        }
        memo[i][j] = max;
        return max;
    }
}
```

**Key Insight:** DFS with memoization. Cycle nahi hai kyunki strictly increasing.

## 11. **Minimum Path Cost in a Grid (Leetcode 2304)**

**Instinct:**  
Grid me (0,0) se (m-1,n-1) tak jaana hai. Move: (i,j) se (i+1, k) where k = 0 to n-1. Cost = grid[i+1][k] + moveCost[grid[i][j]][k]. Minimum path cost.

**Approach:**  
1. Memo[i][j] = min cost from (i,j) to bottom
2. Base: last row → 0 (ya grid value)
3. Try all next columns, add moveCost

```java
class Solution {
    int[][] grid;
    int[][] moveCost;
    int m, n;
    Integer[][] memo;
    
    public int minPathCost(int[][] grid, int[][] moveCost) {
        this.grid = grid;
        this.moveCost = moveCost;
        m = grid.length;
        n = grid[0].length;
        memo = new Integer[m][n];
        
        int minCost = Integer.MAX_VALUE;
        for (int col = 0; col < n; col++) {
            minCost = Math.min(minCost, dfs(0, col));
        }
        return minCost;
    }
    
    private int dfs(int row, int col) {
        if (row == m - 1) return grid[row][col];
        if (memo[row][col] != null) return memo[row][col];
        
        int currentVal = grid[row][col];
        int minNext = Integer.MAX_VALUE;
        
        for (int nextCol = 0; nextCol < n; nextCol++) {
            int next = dfs(row + 1, nextCol);
            minNext = Math.min(minNext, next + moveCost[currentVal][nextCol]);
        }
        
        memo[row][col] = grid[row][col] + minNext;
        return memo[row][col];
    }
}
```

**Key Insight:** Current cell ke value ke basis pe moveCost lookup hota hai.

## 12. **Number of Paths with Max Score (Leetcode 1301)**

**Instinct:**  
Grid me (0,0) se (m-1,n-1) tak jaana hai, right, down, diagonal move allowed. Max score path nikaalni hai aur unki count bhi. 'X' blocked, 'S' start, 'E' end, numbers digits.

**Approach:**  
1. Memo[i][j] = {maxScore, count}
2. Base: (m-1,n-1) → {0,1} if 'E' else invalid
3. Try 3 moves, max score select karo
4. Agar multiple moves same max score, count add karo

```java
class Solution {
    int MOD = 1_000_000_007;
    int m, n;
    String[] board;
    int[][] dirs = {{0,1},{1,0},{1,1}};
    Pair[][] memo;
    
    class Pair {
        int score;
        int count;
        Pair(int s, int c) { score = s; count = c; }
    }
    
    public int[] pathsWithMaxScore(List<String> board) {
        this.board = board.toArray(new String[0]);
        m = this.board.length;
        n = this.board[0].length();
        memo = new Pair[m][n];
        
        Pair res = dfs(0, 0);
        if (res.count == 0) return new int[]{0, 0};
        return new int[]{res.score, res.count % MOD};
    }
    
    private Pair dfs(int i, int j) {
        if (i >= m || j >= n) return new Pair(-1, 0);
        if (board[i].charAt(j) == 'X') return new Pair(-1, 0);
        if (i == m-1 && j == n-1) return new Pair(0, 1);
        if (memo[i][j] != null) return memo[i][j];
        
        int maxScore = -1;
        int totalCount = 0;
        
        for (int[] dir : dirs) {
            Pair next = dfs(i + dir[0], j + dir[1]);
            if (next.score != -1) {
                if (next.score > maxScore) {
                    maxScore = next.score;
                    totalCount = next.count;
                } else if (next.score == maxScore) {
                    totalCount = (totalCount + next.count) % MOD;
                }
            }
        }
        
        int currScore = 0;
        if (board[i].charAt(j) != 'S') {
            currScore = board[i].charAt(j) - '0';
        }
        
        Pair res = new Pair(maxScore == -1 ? -1 : maxScore + currScore, totalCount);
        memo[i][j] = res;
        return res;
    }
}
```

**Key Insight:** Return pair of (maxScore, count). Count tabhi add karo jab score equal ho.

## 13. **Maximum Number of Moves in a Grid (Leetcode 2684)**

**Instinct:**  
Grid me (0, any col) se start karo. Har step me (row+1, col-1/col/col+1) me ja sakte ho, lekin next cell value > current cell value chahiye. Maximum moves possible.

**Approach:**  
1. Memo[i][j] = max moves from (i,j)
2. Try 3 directions, agar value > current then 1 + dp[ni][nj]
3. Base: last row → 0

```java
class Solution {
    int[][] grid;
    int m, n;
    Integer[][] memo;
    
    public int maxMoves(int[][] grid) {
        this.grid = grid;
        m = grid.length;
        n = grid[0].length;
        memo = new Integer[m][n];
        
        int max = 0;
        for (int row = 0; row < m; row++) {
            max = Math.max(max, dfs(row, 0));
        }
        return max;
    }
    
    private int dfs(int i, int j) {
        if (j == n - 1) return 0;
        if (memo[i][j] != null) return memo[i][j];
        
        int maxNext = 0;
        // up-right
        if (i > 0 && grid[i-1][j+1] > grid[i][j]) {
            maxNext = Math.max(maxNext, 1 + dfs(i-1, j+1));
        }
        // right
        if (grid[i][j+1] > grid[i][j]) {
            maxNext = Math.max(maxNext, 1 + dfs(i, j+1));
        }
        // down-right
        if (i < m-1 && grid[i+1][j+1] > grid[i][j]) {
            maxNext = Math.max(maxNext, 1 + dfs(i+1, j+1));
        }
        
        memo[i][j] = maxNext;
        return maxNext;
    }
}
```

**Key Insight:** Strictly increasing condition. j = column, moves = steps taken.

## 14. **Collecting Chocolates (Grid with K moves)**

**Instinct:**  
Grid me (0,0) se (m-1,n-1) tak exactly k steps me jaana hai, collecting maximum chocolates. Har step me right ya down. 3D DP: [i][j][steps].

**Approach:**  
1. Memo[i][j][steps] = max chocolates from (i,j) with remaining steps
2. Base: if i==m-1 && j==n-1 && steps==0 → grid[i][j]
3. Try right and down if within bounds and steps > 0

```java
class Solution {
    int[][][] memo;
    int m, n;
    int[][] grid;
    
    public int maxChocolates(int[][] grid, int k) {
        this.grid = grid;
        m = grid.length;
        n = grid[0].length;
        if (m + n - 2 > k) return -1; // minimum steps needed
        memo = new int[m][n][k+1];
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                Arrays.fill(memo[i][j], -1);
        
        return dfs(0, 0, k);
    }
    
    private int dfs(int i, int j, int steps) {
        if (i >= m || j >= n || steps < 0) return -1;
        if (i == m-1 && j == n-1 && steps == 0) return grid[i][j];
        if (memo[i][j][steps] != -1) return memo[i][j][steps];
        
        int right = dfs(i, j+1, steps-1);
        int down = dfs(i+1, j, steps-1);
        
        int best = Math.max(right, down);
        int res = (best == -1) ? -1 : grid[i][j] + best;
        memo[i][j][steps] = res;
        return res;
    }
}
```

**Key Insight:** Steps count important hai. Minimum steps = (m-1)+(n-1).

## 15. **Minimum Cost to Make at Least One Valid Path (Leetcode 1368)**

**Instinct:**  
Grid me har cell me direction arrow hai (1=right,2=left,3=down,4=up). Tum direction change kar sakte ho with cost 1. Minimum changes to reach (m-1,n-1).

**Approach:**  
1. 0-1 BFS / DP with memo
2. Memo[i][j] = min cost to reach (i,j)
3. Agar current arrow matches desired move → cost 0, else cost 1
4. Use deque for 0-1 BFS

```java
class Solution {
    int m, n;
    int[][] grid;
    int[][] cost;
    int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}}; // right,left,down,up
    
    public int minCost(int[][] grid) {
        this.grid = grid;
        m = grid.length;
        n = grid[0].length;
        cost = new int[m][n];
        for (int i = 0; i < m; i++) Arrays.fill(cost[i], Integer.MAX_VALUE);
        
        dfs(0, 0, 0);
        return cost[m-1][n-1];
    }
    
    private void dfs(int i, int j, int currCost) {
        if (i < 0 || i >= m || j < 0 || j >= n) return;
        if (currCost >= cost[i][j]) return;
        
        cost[i][j] = currCost;
        
        // Try all 4 directions
        for (int d = 0; d < 4; d++) {
            int ni = i + dirs[d][0];
            int nj = j + dirs[d][1];
            int newCost = currCost + (grid[i][j] == d+1 ? 0 : 1);
            dfs(ni, nj, newCost);
        }
    }
}
```

**Key Insight:** DFS with memoization on cost. Agar better cost milta hai toh hi aage badho.

## 16. **Maximum Score from Grid Moves (with negative numbers)**

**Instinct:**  
Grid me (0,0) se (m-1,n-1) tak, right ya down, collect maximum sum. Kuch cells negative ho sakte hain. Simple max path sum.

**Approach:**  
1. Memo[i][j] = max sum from (i,j) to destination
2. Base: destination → grid[i][j]
3. Right and down ka max leke + current

```java
class Solution {
    int[][] grid;
    int m, n;
    Integer[][] memo;
    
    public int maxPathSum(int[][] grid) {
        this.grid = grid;
        m = grid.length;
        n = grid[0].length;
        memo = new Integer[m][n];
        return dfs(0, 0);
    }
    
    private int dfs(int i, int j) {
        if (i == m-1 && j == n-1) return grid[i][j];
        if (memo[i][j] != null) return memo[i][j];
        
        int right = (j+1 < n) ? dfs(i, j+1) : Integer.MIN_VALUE;
        int down = (i+1 < m) ? dfs(i+1, j) : Integer.MIN_VALUE;
        
        int best = Math.max(right, down);
        int res = grid[i][j] + best;
        memo[i][j] = res;
        return res;
    }
}
```

**Key Insight:** Simple 2D DP. Negative numbers handle karne ke liye Integer.MIN_VALUE use karo.

## 17. **Number of Ways to Reach Destination with Obstacles and K moves**

**Instinct:**  
Grid me exactly k moves me (0,0) se (m-1,n-1) tak jaana hai. Right, down, up, left allowed? Usually constraints me only right/down for simplicity.

**Approach:**  
1. Memo[i][j][moves] = ways to reach (i,j) with exactly 'moves' steps
2. Base: (0,0,0) → 1
3. Sum of previous positions (i-1,j), (i,j-1) with moves-1

```java
class Solution {
    int MOD = 1_000_000_007;
    int[][][] memo;
    int m, n;
    
    public int numberOfWays(int[][] grid, int k) {
        m = grid.length;
        n = grid[0].length;
        memo = new int[m][n][k+1];
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                Arrays.fill(memo[i][j], -1);
        
        return dfs(grid, m-1, n-1, k);
    }
    
    private int dfs(int[][] grid, int i, int j, int moves) {
        if (i < 0 || j < 0 || moves < 0) return 0;
        if (i == 0 && j == 0 && moves == 0) return 1;
        if (grid[i][j] == -1) return 0;
        if (memo[i][j][moves] != -1) return memo[i][j][moves];
        
        int ways = (dfs(grid, i-1, j, moves-1) + dfs(grid, i, j-1, moves-1)) % MOD;
        memo[i][j][moves] = ways;
        return ways;
    }
}
```

**Key Insight:** 3D DP (i,j,moves). Top-down approach se destination se start tak reverse mein socho.

## 18. **Minimum Falling Path with Negative Numbers and No Adjacent Columns**

**Instinct:**  
Har row se ek column choose karo, adjacent rows me same column nahi le sakte. Minimum sum. Isme top-2 minimums wala optimization kaam aata hai.

**Approach:**  
1. Memo[row][col] = min sum from (row,col) to bottom
2. Next row ka min find karo except same column
3. Agar multiple mins hain toh track karo

```java
class Solution {
    int[][] matrix;
    int m, n;
    Integer[][] memo;
    
    public int minFallingPathSum(int[][] matrix) {
        this.matrix = matrix;
        m = matrix.length;
        n = matrix[0].length;
        memo = new Integer[m][n];
        
        int minSum = Integer.MAX_VALUE;
        for (int col = 0; col < n; col++) {
            minSum = Math.min(minSum, dfs(0, col));
        }
        return minSum;
    }
    
    private int dfs(int row, int col) {
        if (row == m - 1) return matrix[row][col];
        if (memo[row][col] != null) return memo[row][col];
        
        int minNext = Integer.MAX_VALUE;
        for (int nextCol = 0; nextCol < n; nextCol++) {
            if (nextCol == col) continue;
            minNext = Math.min(minNext, dfs(row + 1, nextCol));
        }
        
        int res = matrix[row][col] + minNext;
        memo[row][col] = res;
        return res;
    }
}
```

**Key Insight:** O(m*n^2) with memoization. Optimized O(m*n) ke liye top-2 minima maintain karo.

## 19. **Maximum Non-Negative Product in a Grid (Leetcode 1594)**

**Instinct:**  
(0,0) se (m-1,n-1) tak right/down moves. Product maximize karna hai, but product negative ho sakta hai. Isliye dono max aur min track karna hoga (kyunki negative * negative = positive).

**Approach:**  
1. Memo[i][j] = {maxProduct, minProduct}
2. Base: (0,0) → {grid[0][0], grid[0][0]}
3. Right and down se max/min compute karo
4. Return max at (m-1,n-1)

```java
class Solution {
    int MOD = 1_000_000_007;
    long[][][] memo;
    int m, n;
    
    public int maxProductPath(int[][] grid) {
        m = grid.length;
        n = grid[0].length;
        memo = new long[m][n][2]; // 0=max, 1=min
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                Arrays.fill(memo[i][j], Long.MAX_VALUE);
        
        long[] res = dfs(grid, 0, 0);
        return res[0] < 0 ? -1 : (int)(res[0] % MOD);
    }
    
    private long[] dfs(int[][] grid, int i, int j) {
        if (i == m-1 && j == n-1) {
            return new long[]{grid[i][j], grid[i][j]};
        }
        if (memo[i][j][0] != Long.MAX_VALUE) return memo[i][j];
        
        long maxVal = Long.MIN_VALUE;
        long minVal = Long.MAX_VALUE;
        
        // right
        if (j+1 < n) {
            long[] right = dfs(grid, i, j+1);
            long r1 = grid[i][j] * right[0];
            long r2 = grid[i][j] * right[1];
            maxVal = Math.max(maxVal, Math.max(r1, r2));
            minVal = Math.min(minVal, Math.min(r1, r2));
        }
        
        // down
        if (i+1 < m) {
            long[] down = dfs(grid, i+1, j);
            long d1 = grid[i][j] * down[0];
            long d2 = grid[i][j] * down[1];
            maxVal = Math.max(maxVal, Math.max(d1, d2));
            minVal = Math.min(minVal, Math.min(d1, d2));
        }
        
        memo[i][j][0] = maxVal;
        memo[i][j][1] = minVal;
        return new long[]{maxVal, minVal};
    }
}
```

**Key Insight:** Negative numbers handle karne ke liye max aur min dono track karo.

## 20. **Collect Maximum Coins with Two Robots (Grid with two pickers)**

**Instinct:**  
Do robots ek saath (0,0) se (m-1,n-1) tak ja rahe hain. Har cell ko sirf ek baar count karna hai. Cherry pickup jaisa hi problem hai.

**Approach:**  
1. Memo[r1][c1][r2] = max coins with robot1 at (r1,c1) and robot2 at (r2,c2)
2. c2 = r1+c1-r2 (same as cherry pickup)
3. Try all 4 combinations of moves
4. Agar same cell hai toh ek baar add karo, warna dono ka sum

```java
class Solution {
    int[][] grid;
    int m, n;
    Integer[][][] memo;
    
    public int maxCoins(int[][] grid) {
        this.grid = grid;
        m = grid.length;
        n = grid[0].length;
        memo = new Integer[m][n][m];
        return dfs(0, 0, 0);
    }
    
    private int dfs(int r1, int c1, int r2) {
        int c2 = r1 + c1 - r2;
        if (r1 >= m || c1 >= n || r2 >= m || c2 >= n) return Integer.MIN_VALUE;
        if (r1 == m-1 && c1 == n-1) return grid[r1][c1];
        if (memo[r1][c1][r2] != null) return memo[r1][c1][r2];
        
        int coins = grid[r1][c1];
        if (r1 != r2 || c1 != c2) coins += grid[r2][c2];
        
        int maxNext = Math.max(
            Math.max(dfs(r1+1, c1, r2+1), dfs(r1+1, c1, r2)),
            Math.max(dfs(r1, c1+1, r2+1), dfs(r1, c1+1, r2))
        );
        
        int res = coins + maxNext;
        memo[r1][c1][r2] = res;
        return res;
    }
}
```

**Key Insight:** Cherry pickup ka variation. 3D DP (r1,c1,r2) with c2 derived.

## 🎯 Key Takeaways for 2D Grid DP (Memoization)

1. **States identify karo:** Usually `(i, j)` sufficient hai. Kabhi kabhi extra state like `steps`, `holding`, `count` chahiye.

2. **Base case:** Boundary conditions aur destination cell handle karo.

3. **Direction of recursion:** Top-down me (0,0) se call karo aur (m-1,n-1) base case.

4. **Invalid states:** `Integer.MIN_VALUE` ya `-1` use karo, INF overflow se bachne ke liye `Integer.MAX_VALUE/2`.

5. **Multiple returns:** Kabhi kabhi (max, count) ya (max, min) dono return karne padte hain.

6. **Space complexity:** Memoization ka space O(m*n) ya O(m*n*k) hota hai.

7. **Optimization:** Agar O(m*n^2) chal raha hai, socho top-2 minima ya left-right sweep lagao.
